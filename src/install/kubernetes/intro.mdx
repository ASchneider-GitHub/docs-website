The New Relic Kubernetes integration gives you full observability into the health and performance of your infrastructure. This agent collects telemetry data from your cluster using several New Relic integrations such as the [Kubernetes events integration](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/kubernetes-events-integration/), the [Prometheus agent](/docs/infrastructure/prometheus-integrations/install-configure-prometheus-agent/install-prometheus-agent/), the [nri-kubernetes](https://github.com/newrelic/nri-kubernetes), and the [New Relic Logs Kubernetes plugin](/docs/logs/forward-logs/kubernetes-plugin-log-forwarding/).


If you're going to install the New Relic Kubernetes integration on Windows or EKS Fargate, see the [Install Kubernetes on Windows](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-windows) and [Install Kubernetes on AWS EKS Fargate](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-eks-fargate) documents.

Read on to install the integration and to see what data we collect.
