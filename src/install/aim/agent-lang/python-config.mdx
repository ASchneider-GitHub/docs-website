You installed the Python agent in the last step, but you still need to apply certain configurations so the agent can collect AI-specific data. 

We recommend applying these configurations from `newrelic.ini` config file. If you have a more complex set up with multiple environments, you can configure with environment variables.

1. To enable AI Monitoring:

    <CollapserGroup>
        <Collapser
            id="enable-ai-file"
            title="Enable with config file"
        >

    ```ini
    ai_monitoring.enabled = true
    ```   

        </Collapser>
        <Collapser
            id="enable-ai-enviro"
            title="Enable with environment variables"
        >

    ```bash
    NEW_RELIC_AI_MONITORING_ENABLED=true
    ```

        </Collapser>
    </CollapserGroup>

2. Set the agent to collect the max number of spans from your AI events:

    <CollapserGroup>
        <Collapser
            id="set-max-spans-file"
            title="Set max spans with config file"
        >
            
        ```ini
            event_harvest_config.harvest_limits.span_event_data
        ```  

        </Collapser>
        <Collapser
            id="set-max-spans-enviro"
            title="Set max spans with environment variables"
        >
        ```bash
            NEW_RELIC_SPAN_EVENTS_MAX_SAMPLES_STORED
        ```
            </Collapser>
    </CollapserGroup>

You can make further adjustments to what AI data the Python agent sends to New Relic in our [AI Monitoring configuration](/docs/ai-observability/configure-for-aim/) doc. For example, you can adjust how many AI events the agent collects.  

If you want to make additional configurations to how your Python agent collects data about your Python app, see our [Python configuration](/docs/apm/agents/python-agent/configuration/python-agent-configuration) doc. For example, you can enable log forwarding for your Python app.
