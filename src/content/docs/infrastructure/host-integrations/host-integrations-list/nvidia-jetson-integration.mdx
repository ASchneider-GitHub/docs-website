## Install the infrastructure agent [#infra-install]

To use the NVIDIA Jetson integration, you need to also [install the infrastructure agent](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic/) on the same host. The infrastructure agent monitors the host itself, while the integration you'll install in the next step extends your monitoring with NVIDIA Jetson-specific data.
    </Step>
     <Step>
## Save tegrastats to a file
Run the following command to save Jetson metrics to the tegrastats file at 5-second intervals
```shell
sudo tegrastats --interval 5000 --logfile tegrastats
```    
     </Step>
    <Step>

## Enable the NVIDIA Jetson integration with nri-flex

To set up the NVIDIA Jetson integration, follow these steps:

1. Create and edit file named `nvidia-jetson-config.yml` in the integrations directory:

  ```shell
  sudo nano /etc/newrelic-infra/integrations.d/nvidia-jetson-config.yml
  ```

2. Add the following snippet to your `nvidia-jetson-config.yml` file to enable the agent to capture NVIDIA Jetson:
  ```yml
  integrations:
    - name: nri-flex
      config:
        name: JestonMetrics
        apis:
          - event_type: tegrastats
            commands:
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [ram_used, ram_total]
                regex_match: true
                split_by: "RAM (\\d+)/(\\d+)"
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [lfb_num, lfb_size]
                regex_match: true
                split_by: "\\(lfb (\\d+)x(\\d+)MB\\)"
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [swap_used, swap_total, swap_cached]
                regex_match: true
                split_by: "SWAP (\\d+)/(\\d+) \\(cached (\\d+)MB\\)"    
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [cpu1_usage, cpu1_freq]
                regex_match: true
                split_by: "CPU \\[(\\d+)%@(\\d+)" 
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [cpu2_usage, cpu2_freq]
                regex_match: true
                split_by: ",(\\d+)%@(\\d+)" 
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [cpu3_usage, cpu3_freq]
                regex_match: true
                split_by: ",(\\d+)%@(\\d+)" 
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [cpu4_usage, cpu4_freq]
                regex_match: true
                split_by: ",(\\d+)%@(\\d+)\\]"       
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [emc_freq_pct, emc_freq_val]
                regex_match: true
                split_by: "EMC_FREQ (\\d+)%@(\\d+)"
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [gr3d_freq_pct, pll_temp]
                regex_match: true
                split_by: "GR3D_FREQ (\\d+)%.+PLL@(\\d+)"  
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [cpu_temp, pmic_temp, gpu_temp]
                regex_match: true
                split_by: "CPU@(\\d+\\.?\\d*)C? PMIC@(\\d+\\.?\\d*)C? GPU@(\\d+\\.?\\d*)C?"
              - run: head -n 3 path/to/tegrastats
                split: horizontal
                set_header: [ao_temp, thermal_temp]
                regex_match: true
                split_by: "AO@(\\d+\\.?\\d*)C? thermal@(\\d+\\.?\\d*)C?"  
  ```
    </Step>
    <Step>


## Restart the New Relic infrastructure agent

Restart your infrastructure agent:

  ```shell
  sudo systemctl restart newrelic-infra.service
  ```

In a couple of minutes, your application will begin sending metrics to [one.newrelic.com](https://one.newrelic.com).        
    </Step>
    <Step>
## Find your data 

You can use our pre-built dashboard template to monitor your NVIDIA Jetson application metrics. Follow these steps to use our pre-built dashboard template:

1. From [one.newrelic.com](https://one.newrelic.com), go to the <DoNotTranslate>**+ Add data**</DoNotTranslate> page
2. Click on <DoNotTranslate>**Dashboards**</DoNotTranslate>
3. In the search bar, type `NVIDIA Jetson`
4. The NVIDIA Jetson dashboard should appear. Click on it to install it

Your NVIDIA Jetson dashboard is considered a custom dashboard and can be found in the <DoNotTranslate>**Dashboards**</DoNotTranslate> UI. For docs on using and editing dashboards, see [our dashboard docs](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Here is an example NRQL query to view the GPU temperature:

```sql
SELECT latest(gpu_temp) as 'GPU temperature (C)' from jetsonTegrastats 
```
