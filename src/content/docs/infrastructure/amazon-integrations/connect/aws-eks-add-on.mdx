## This document shows the steps required to use EKS Add-On from AWS Marketplace.
#### **NOTE : This add-on is currently only supported on x86 instance types, with support for ARM64 coming soon**

## Prerequisites
1. [Amazon Elastic Kubernetes Service (EKS)](https://docs.aws.amazon.com/eks/latest/userguide/what-is-eks.html) Cluster. See the userguide [here](https://docs.aws.amazon.com/eks/latest/userguide/create-cluster.html)
2. A valid New Relic account. You can subscribe to New Relic from AWS Marketplace with Free Tier. See details [here](https://aws.amazon.com/marketplace/pp/prodview-ov56chowabeb4?sr=0-3&ref_=beagle&applicationId=AWS-Marketplace-Console)
3. Subscribe to [New Relic Kubernetes Integration](https://aws.amazon.com/marketplace/pp/prodview-gcywa6keq2ajy?applicationId=AWS-Marketplace-Console&ref_=beagle&sr=0-5), available for free
4. Login to [New Relic](https://one.newrelic.com/) and generate a [License Key](https://one.newrelic.com/api-keys)

## Provision New Relic Kubernetes Operator EKS Add-On via AWS console
1. Navigate to the EKS Cluster > Add-Ons > Get More add-ons
    <img
      title="eksAddOnTab.png"
      alt="EKS Add On Tab."
      src={eksAddOnTab}
    />
2. Under AWS Marketplace add-on, search, browse or navigate to New Relic Kubernetes Integration, select the checkbox and click Next
    <img
      title="eksAddOnMarketplace.png"
      alt="EKS Add On Marketplace."
      src={eksAddOnMarketplace}
    />
3. Select the latest default version or choose an alternate version, if required, and click Next
    <img
      title="eksAddOnMarketplaceConfig.png"
      alt="EKS Add On Marketplace Config"
      src={eksAddOnMarketplaceConfig}
    />
4. Click Create

5. Within a few minutes, New Relic's Kubernetes Operator should be up and running. Please view the configuration steps below to collect and ship observability data from your EKS Cluster to New Relic.


## Provision NewRelic EKS Add-On via AWS Command Line Interface (CLI)

1. Execute the following command to install the NewRelic Add-On

   ```
   aws eks create-addon --cluster-name REPLACE_ME --addon-name new-relic_kubernetes-operator
   ```

  where --cluster-name : Name of your EKS Cluster

2. Expected Output
   Output will be similar to the following:

## New Relic Kubernetes Operator Configuration

You may confirm that the Kubernetes Operator is running by executing ** kubectl get all -n newrelic ** at CLI.

To start collecting metrics and ship to New Relic, use the following sample Custom Resource Definition (CRD)

1. Create crd.yml file using below definitions.
    ```
    apiVersion: newrelic.com/v1alpha1
    kind: NRIBundle
    metadata:
      name: nribundle-minimal
    spec:
      global:
        cluster: "<cluster-name>"
        licenseKey: "<licensekey>"
        lowDataMode: true
      newrelic-infrastructure:
        enabled: true
        privileged: true
    ```
where ** spec.global.cluster ** : Name of your EKS Cluster and 
**spec.global.licensekey**: Your New Relic License Key that you generated at step-4 in the Prerequisites section above.


2. after that apply crd using below command - 
    ```
    kubectl apply -f crd.yml -n newrelic
    ```
