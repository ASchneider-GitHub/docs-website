The [New Relic Ruby agent](/docs/agents/ruby-agent/getting-started/new-relic-ruby) automatically instruments [Roda](https://roda.jeremyevans.net/) 3.19.0 and higher. 

<Callout variant="important">
  <DNT>Roda</DNT> instrumentation requires Ruby agent version 9.4.0 or higher.
</Callout>

## Getting started

To set up your <DNT>Roda</DNT> app with New Relic:

1. Add the `newrelic_rpm` gem to Gemfile and bundle.
   ```
   gem 'newrelic_rpm'
   ```
2. In your <DNT>Roda</DNT> app, below the <DNT>Roda</DNT> <DNT>**require**</DNT> directive, add `require 'newrelic_rpm'`. For example:

  ```
  require 'roda'
  require 'newrelic_rpm'

  class App < <DNT>Roda</DNT>
    route do |r|
      r.root do
        "Hello World!"
      end
    end
  end
  ```

## Ignoring routes

Ruby agent versions 9.6.0 or higher support ignoring certain routes. To specify these values, use the same style of routes that you use to define your <DNT>Roda</DNT> application. For example, to ignore a `hello_world` route in a <DNT>Roda</DNT> app, declare `newrelic_ignore '/hello_word'` outside the `route` block:

```
newrelic_ignore '/hello_world'

route do |r|
  r.on '/hello_world' do
    "Hello World!"
  end
end
```

If you want an entire application to be ignored (for example, in a mounted application), call `newrelic_ignore` without parameters:

```
newrelic_ignore
```

Additionally, `newrelic_ignore_apdex` and `newrelic_ignore_enduser` are supported.

* The `newrelic_ignore_apdex` call will exclude a given route from consideration in overall Apdex calculations.
* The `newrelic_ignore_enduser` call will prevent automatic injection of the page load timing JavaScript when a route is rendered.

<Callout variant="important">
  All newrelic_ignore* methods must be called outside the `route` block.
</Callout>

## Rack instrumentation

As a Rack-based framework, <DNT>Roda</DNT> instrumentation requires Rack instrumentation to work properly. Rack instrumentation is enabled by default.
