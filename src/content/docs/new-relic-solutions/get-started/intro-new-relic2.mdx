Our all-in-one platform brings all that telemetry data to one place. It enables you to set alerts and detect anomalies. # Learn how it works

The first step in observability requires collecting lots of performance data, known as telemetry data. Our observability platform gathers data from our services using agents. A process called instrumentation integrates agents into a service. An agent is a file or a few lines of code that collects data on your service and sends it to our platform for analysis.

The agent keeps track of your service's performance by monitoring metrics such as response times and error rates. By analyzing this data, you can gain insights into the functioning of your service and identify areas for improvement. An agent is first installed in your environment with a default configuration. The process of installation, setup, and instrumentation can vary. It depends on the service you want to collect data from.

These are the agents at New Relic:

  * **APM agent**: For server-side applications

  * **Browswer agent**: For browser applications

  * **Infrastructure agents**: For hosts and on-host integrations

  * **Mobile agent**: For mobile applications

Besides these agents, we also support open-source integrations. They report data from open standards for instrumentation. These standards include OpenTelemetry and tools like DropWizard, Prometheus, and more.

New Relic provides visibility through a library of capabilities. Our many capabilities will help you monitor, troubleshoot, and improve your digital performance. Here's an explanation of New Relic's key capabilities.

Alerts help you identify issues that look unusual in your applications and code. New Relic includes some alerts when you install it and you can also create your alerts. You receive alerts through tools like PagerDuty and ServiceNow. Let us help you know what's critical, manage the suppressed noise, and mitigate alert fatigue.
