## Grid

Grid is a 2-dimensional layout system, meaning that it can handle both columns and rows, that helps you design grid-based user interfaces, unlike `<Stack>` which is largely a 1-dimensional system.

### Usage

```
import { Grid } from 'nr1'
```

### Props

|                                    |                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `children` <h5>REQUIRED</h5> <h5>node</h5> | Grid items to display.Up to 12 items are allowed.                                                                                                                                                                                                                                                                                                                                                                                              |
| `className` <h5>string</h5>            | Appends class names to the component.Should be used only for positioning and spacing purposes.                                                                                                                                                                                                                                                                                                                                                 |
| `fullHeight` <h5>boolean</h5>          | Expands the grid to occupy all available height.                                                                                                                                                                                                                                                                                                                                                                                               |
| `fullWidth` <h5>boolean</h5>           | Expands the grid to occupy all available width.                                                                                                                                                                                                                                                                                                                                                                                                |
| `gapType` <h5>enum</h5>                | Size of the gap between columns and rows.<OptionReference>Grid.GAP_TYPE.MEDIUMGrid.GAP_TYPE.SMALL</OptionReference>                                                                                                                                                                                                                                                                                                                 |
| `spacingType` <h5>enum[]</h5>         | Spacing property. Spacing is defined as a tuple of zero to four values, which follow the same conventions as CSS properties like `margin` or `padding`. To omit a value, use `SPACING_TYPE.OMIT`.<OptionReference array>Grid.SPACING_TYPE.EXTRA_LARGEGrid.SPACING_TYPE.LARGEGrid.SPACING_TYPE.MEDIUMGrid.SPACING_TYPE.NONEGrid.SPACING_TYPE.OMITGrid.SPACING_TYPE.SMALL</OptionReference> |
| `style` <h5>object</h5>                | Inline style for custom styling.Should be used only for positioning and spacing purposes.                                                                                                                                                                                                                                                                                                                                                      |
| `testId` <h5>string</h5>               | Adds a `data-test-id` attribute. Use it to target the component in unit and E2E tests.For a test id to be valid, prefix it with your nerdpack id, followed up by a dot.For example, `my-nerdpack.some-element`.**Note:** You might not see `data-test-id` attributes as they are removed from the DOM, to debug them pass a `e2e-test` query parameter to the URL.                                                                             |
