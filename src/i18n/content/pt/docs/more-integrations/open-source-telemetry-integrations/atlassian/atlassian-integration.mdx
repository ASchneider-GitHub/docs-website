New Relic oferece uma integração com o Atlassian Jira que possibilita o recurso de aba incidente dentro do Jira. Este recurso sincroniza todos os seus problemas e informações de incidentes New Relic com o Atlassian. Esses dados aparecem no Jira na guia incidente e permitem conectá-los aos problemas do Jira para melhorar o fluxo de trabalho da interrupção. As seções abordam como você configura essa integração e as práticas recomendadas de uso.

<Callout title="PREVIEW recurso">
  Este recurso ainda está em versão prévia, mas encorajamos você a experimentá-lo!
</Callout>

## Compartilhamento de dados [#data-sharing]

Depois de instalar a integração, você permite que o New Relic for Jira execute estas ações:

* Leia entidade, incidente e problemas do Jira.
* Escreva entidade, incidente e problemas para Jira.
* Exclua entidade, incidente e problemas do New Relic e do Jira para espelhar o relatório de dados com cada entidade.

<Callout variant="caution">
  Essa integração exporá seus problemas New Relic e informações de incidentes a todos os usuários no espaço de trabalho Jira relacionado. O usuário Jira neste espaço de trabalho pode não ter acesso ao New Relic e, mesmo que tenha, pode não ter acesso às mesmas contas New Relic que seu usuário chave de API.

  Se você tiver informações de conta da New Relic para ocultar, use um [usuário de integração](/docs/more-integrations/best-practices-integration/#integration-user-leverage) que não tenha acesso às suas contas confidenciais.
</Callout>

## instalação [#installation]

Siga estas etapas para integrar o New Relic ao Atlassian Jira.

<Steps>
  <Step>
    ## Instale o aplicativo de mercado New Relic [#install-marketplace]

    Você precisa instalar a New Relic Errors Inbox para Jira para manter seus dados sincronizados entre New Relic e o Atlassian. Lembre-se de que você deve ter permissão de administrador do site Jira para instalar o aplicativo Marketplace. Acesse [Atlassian Marketplace](https://marketplace.atlassian.com/apps/1227457/new-relic-errors-inbox-for-jira?hosting=cloud&tab=overview) e clique no botão **Install** .

    <Callout variant="tip">
      Se você não tiver permissão de administrador do site, poderá solicitá-la ao seu administrador. Recomendamos que você entre em contato com o administrador do site Jira para concluir o processo de instalação do aplicativo.
    </Callout>
  </Step>

  <Step>
    ## Crie uma chave de API New Relic [#create-new-relic-key]

    Após a instalação do aplicativo New Relic Atlassian Marketplace, você precisa criar uma [chave de API New Relic](/docs/apis/intro-apis/new-relic-api-keys/). O administrador do site Jira e o administrador do New Relic decidirão sobre as contas expostas ao Jira. O usuário que tiver o acesso correto à conta New Relic poderá gerar uma chave de API New Relic . A chave de API New Relic tem acesso às mesmas contas do usuário que as gerou.

    Crie e gerencie sua chave de API na página [New Relic de interface de chave de API](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) ou na página de log [API token](https://id.atlassian.com/manage-profile/security/api-tokens) da Atlassian e clique em **Create API token**.
  </Step>

  <Step>
    ## Aplicar uma chave de API New Relic [#apply-new-relic-key]

    Depois de criar uma chave de API, você poderá aplicá-la à primeira página do aplicativo de conexão New Relic.

    Cole seu token da API New Relic no aplicativo New Relic Atlassian Marketplace e clique em **Connect**.

    <img
      width="50%;"
      title="Apply a New Relic API key"
      alt="Apply a New Relic API key"
      src={opentelemetryAtlassianNRapiToken}
    />
  </Step>

  <Step>
    ## Selecione seus serviços New Relic [#select-nr-services]

    Depois de criar uma chave de API no aplicativo New Relic Atlassian Marketplace, você verá uma lista de todas as entidades associadas às contas nessa chave de API. Abra a página de gerenciamento de incidentes do Jira e selecione toda a entidade que deseja expor no Jira.

    <img
      title="Jira incident management page"
      alt="Jira incident management page"
      src={opentelemetryJiraIncidentManagement}
    />
  </Step>

  <Step>
    ## Veja os problemas e incidentes New Relic no Atlassian Jira [#see-issues-incidents]

    Depois de selecionar sua entidade New Relic , o aplicativo começa a sincronizar os dados do New Relic com o Atlassian Jira. Abra a guia incidente para ver uma lista de incidentes detectados.
  </Step>
</Steps>

## Habilite a aba incidente [#enable-incidents-tab]

Pode ser necessário vincular à documentação do Atlassian. Ele orienta o administrador do site Jira na ativação da guia Incidente e na atribuição dela a um espaço de trabalho. Há uma chance de que eles sejam ativados por padrão e você não precisará desta seção.

## Remover uma conexão entre New Relic e Atlassian Jira [#remove-connection]

Você pode interromper a sincronização de dados do New Relic a qualquer momento e remover os dados sincronizados. Como administrador do site JIRA, você pode fazer isso na página de configuração do Atlassian Marketplace. Você pode selecionar sua organização conectada e clicar em **Delete**. Após a conclusão, você não verá mais as informações de incidentes e problemas New Relic presentes na guia de incidentes no Jira.

## Conectar-se a uma organização ou usuário diferente [#different-conexion]

A qualquer momento, você pode se conectar à organização New Relic errada ou pode querer ajustar o acesso à conta que sua chave de API fornece. Você pode fazer isso na página do Atlassian Marketplace. Ao clicar em **Connect another New Relic organization**, você precisará fornecer uma chave de API New Relic. Fornecer uma nova chave excluirá sua conexão atual e incluirá todos os dados sincronizados com o Atlassian. Em seguida, um novo processo de sincronização será iniciado com o acesso à conta e à organização da nova chave de API.
