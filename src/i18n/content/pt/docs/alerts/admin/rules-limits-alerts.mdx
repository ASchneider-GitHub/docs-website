## Consulta de alerta NRDB com pontos de dados correspondentes por minuto [#query-limit]

O limite de condição do alerta `Matched data points per minute` se aplica à taxa total de [pontos de dados](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/understand-technical-concepts/streaming-alerts-key-terms-concepts) correspondentes para a consulta de alerta em uma [conta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) New Relic .

If this limit is exceeded, you won&apos;t be able to create or update conditions for the impacted account until the rate goes below the limit. Existing alert conditions **aren&apos;t** affected.

Você pode ver seus pontos de dados correspondentes e qualquer incidente de limite na [interface de limites](/docs/data-apis/manage-data/view-system-limits).

Para entender quais condições estão levando ao maior número de taxas de transferência, você pode realizar uma consulta como:

```sql
FROM NrAiSignal
SELECT sum(aggregatedDataPointsCount) AS 'alert matched data points'
FACET conditionId
```

Algumas dicas sobre como otimizar seus pontos de dados correspondentes:

* Se você estiver usando [janelas deslizantes](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions/#sliding-window-aggregation), observe que isso pode aumentar significativamente o número de pontos de dados. Para diminuir o número de pontos de dados, você pode usar uma duração de agregação mais longa.
* Use cláusulas `WHERE` para reduzir a quantidade de dados que estão sendo alertados. Usar `WHERE` em vez de `FACET` pode produzir alertas mais eficientes em alguns casos.
* Combine alerta semelhante. Se você tiver várias condições de alerta semelhantes, considere agrupá-las com filtros combinados.

Para solicitar um aumento de limite, fale com seu representante de conta New Relic.

Observe que o uso [de janelas deslizantes](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/create-smoother-charts-sliding-windows) pode aumentar significativamente o número de pontos de dados. Considere usar uma duração mais longa de agregação de janela deslizante para reduzir o número de pontos de dados produzidos.

## Alert query scan operations per minute [#query-scan-limit]

The alert condition `Alert query scan operations per minute` limit applies to the total rate of query scan operations on ingested events. A query scan operation is the work performed by the New Relic pipeline to match ingested events to alert queries registered in a New Relic [account](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure).

If this limit is exceeded, you won&apos;t be able to create or update conditions for the impacted account until the rate goes below the limit. Existing alert conditions **aren&apos;t** affected.

You can see your query scan operations and any limit incidents in the [limits UI](/docs/data-apis/manage-data/view-system-limits).

When matching events to alert queries, all events from the [data type](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/#what-you-can-query) that the query references must be examined. Here are a few common ways to have fewer events in a given data type (which will decrease the alert query scan operations):

* When alerting on logs data, use [log partitions](/docs/tutorial-manage-large-log-volume/organize-large-logs/) to limit which logs are being scanned for alert queries.

* When alerting on custom events, break up larger custom event types.

* Use custom events instead of alerting on transaction events.

* [Create metrics](/docs/data-apis/convert-to-metrics/analyze-monitor-data-trends-metrics/) to aggregate data.

* Use [metric timeslice queries](/docs/data-apis/understand-data/metric-data/query-apm-metric-timeslice-data-nrql/) when possible instead of alerting on transaction events.

In addition to the above tips, cleaning up any unused or unneeded alert queries (alert conditions) will decrease the number of query scan operations.

Para solicitar um aumento de limite, fale com seu representante de conta New Relic.
