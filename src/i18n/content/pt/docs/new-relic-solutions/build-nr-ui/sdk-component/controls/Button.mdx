Botões são usados em todo o produto para fornecer ao usuário uma maneira de concluir uma ação. Tente não abusar dos botões em sua experiência. Isso ajudará o usuário a saber exatamente que ação você gostaria que ele realizasse. Nunca use mais de um botão principal em sua experiência.

Capacidades primárias — os botões podem

* realizar uma ação
* executar múltiplas ações, se vários botões estiverem presentes.

### Uso

```js
import { Button } from 'nr1'
```

### Exemplos

#### Básico

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={
Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={
Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `ariaControls` <h5>corda</h5>
      </td>

      <td>
        Passe a string `id` do elemento que o `Button` controla quando ele é usado para expandir ou abrir um painel. Use-o junto com `ariaExpanded`.

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
          <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
              Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
              Content
            </div>
          </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded` <h5>boleano</h5>
      </td>

      <td>
        Use-o para indicar se o elemento que o `Button` controla está expandido ou não. É recomendado verificar também os exemplos `ariaControls` e `ariaHasPopup` .
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup` <h5>boolean|string</h5>
      </td>

      <td>
        Use-o para indicar que `Button` alterna um elemento de sobreposição interativo. O valor pode ser:

        * `false` (padrão) indica que o elemento não possui um pop-up.
        * `true` indica que o pop-up tem uma função `menu`
        * `"menu"` indica que o pop-up tem uma função `menu`
        * `"listbox"` indica que o pop-up tem uma função `listbox`
        * `"tree"` indica que o pop-up tem uma função `tree`
        * `"grid"` indica que o pop-up tem uma função `grid`
        * `"dialog"` indica que o pop-up tem uma função `dialog` Leia [as especificações WAI ARIA desse atributo](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup) para saber qual delas é mais adequada para seu caso de uso.

        ```js
        const { modalHidden } = this.state;

        return (
          <>
            <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
            >
              Open modal
            </Button>
            <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
            />
          </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaLabel` <h5>corda</h5>
      </td>

      <td>
        Utilize-o para descrever melhor o contexto da ação do componente ou em botões que exibem apenas um ícone para o usuário em leitores de tela.

        ```js
        <Button
          iconType={Button.ICON_TYPE.INTERFACE__OPERATIONS__SEARCH}
          ariaLabel="Search for entities"
        />;
        ```
      </td>
    </tr>

    <tr>
      <td>
        `children` <h5>nó</h5>
      </td>

      <td>
        Conteúdo a ser renderizado dentro do botão.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>corda</h5>
      </td>

      <td>
        Acrescenta nomes de classes ao componente. Deve ser usado apenas para fins de posicionamento e espaçamento.
      </td>
    </tr>

    <tr>
      <td>
        `disabled` <h5>boleano</h5>
      </td>

      <td>
        Use o estado desativado para um botão antes de um usuário preencher os campos obrigatórios de um formulário ou quando um usuário precisar concluir alguma outra tarefa antes que o botão possa ser ativado.
      </td>
