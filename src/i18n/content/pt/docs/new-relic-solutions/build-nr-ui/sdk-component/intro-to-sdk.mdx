Para ajudá-lo a criar o aplicativo New Relic , fornecemos os componentes do SDK New Relic , que são usados para criar aplicativos New Relic .

Aqui damos uma introdução aos tipos de chamada de API e componentes do SDK. O SDK fornece tudo que você precisa para construir seus Nerdlets, criar visualizações e buscar dados da New Relic ou de terceiros.

## Componentes do SDK [#components]

Você pode encontrar os componentes do SDK no pacote do módulo Node chamado `nr1`, que você obtém ao instalar a [CLI do New Relic](/docs/new-relic-solutions/tutorials/new-relic-cli/#install-the-new-relic-cli). Os componentes `nr1` são divididos em diversas categorias:

* Componentes de interface
* Componentes do gráfico
* Componentes de consulta e armazenamento
* API da plataforma

### Componentes de interface [#ui-components]

A categoria **UI components** do SDK contém componentes de interface React, incluindo:

* **Componentes de texto**: esses componentes fornecem elementos básicos de fonte e título. Isso inclui `HeadingText` e `BlockText`.

* **Componentes de layout**: esses componentes fornecem controle sobre o layout e ajudam a construir designs de layout complexos sem ter que lidar com CSS. Os componentes de layout incluem:

  * `Grid` e `GridItem`: para organizar conteúdo de página mais complexo e de maior escala em linhas e colunas.
  * `Stack` e `StackItem`: para organizar conteúdo de página mais simples e de menor escala, em coluna ou linha.
  * `Tabs` e `TabsItem`: agrupa vários conteúdos relacionados em seções ocultáveis separadas.
  * `List` e `ListItem`: para fornecer um esqueleto básico de listas virtualizadas.
  * `Card`, `CardHeader` e `CardBody`: usados para agrupar conceitos e tarefas semelhantes.

* **Componentes de formulário**: Esses componentes fornecem os blocos de construção básicos para interagir com a interface. Isso inclui [`Button`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/button), [`TextField`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/text-field), [`Dropdown`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown), [`DropdownItem`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown-item), [`Checkbox`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/checkbox), [`RadioGroup`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio-group) e [`Radio`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio).

* **Componentes de feedback**: use esses componentes para fornecer feedback ao usuário sobre as ações que ele realizou. Estes incluem: `Spinner` e `Toast`.

* **Componentes sobrepostos**: use esses componentes para exibir informações e opções contextuais na forma de uma visualização secundária adicional que aparece acima de outro conteúdo na tela quando uma ação ou evento é acionado. Eles podem exigir interação do usuário (como modais) ou ser aumentados (como uma dica de ferramenta). Estes incluem: `Modal` e `Tooltip`.

<Callout variant="important">
  Os componentes com o sufixo `Item` só podem operar como filhos diretos desse nome sem o sufixo. Por exemplo: `GridItem` só deve ser encontrado como filho de `Grid`.
</Callout>

### Componentes do gráfico [#chart-components]

A categoria **Charts** do SDK contém componentes que representam diferentes tipos de gráficos. O componente `ChartGroup` ajuda um grupo de gráficos relacionados a compartilhar dados e a se alinhar.

Alguns componentes do gráfico podem realizar [consultasNRQL ](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/)por conta própria; alguns aceitam um conjunto personalizado de dados.

### Componentes de consulta e armazenamento [#query-storage]

A categoria **Query components** contém componentes para buscar e armazenar dados do New Relic.

A principal forma de buscar dados é com NerdGraph, nosso endpoint GraphQL. Isso pode ser consultado usando `NerdGraphQuery`. Para simplificar o uso da consulta NerdGraph, disponibilizamos alguns componentes com consulta pré-definida. Consulte nossos documentos sobre [o NerdGraph](/collect-data/get-started-nerdgraph-api-explorer) para obter mais informações.

Também fornecemos armazenamento para armazenar pequenos conjuntos de dados, como dados de configurações de configuração ou dados específicos do usuário. Para saber mais sobre isso, consulte [NerdStorage](/docs/new-relic-solutions/build-nr-ui/nerdstorage).

### API da plataforma

Os componentes API da plataforma do SDK permitem que seu aplicativo interaja com diferentes partes da plataforma New Relic , lendo e gravando o estado de e para a URL, definindo a configuração, etc. Eles são divididos nestas categorias:

* `PlatformStateContext`: fornece acesso de leitura às variáveis de estado do URL da plataforma. Por exemplo `timeRange` no [seletor de hora](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker).
* `navigation`: Um objeto que permite a manipulação programática da navegação no New Relic. Exemplo: abrindo um novo Nerdlet.
* `NerdletStateContext`: fornece acesso de leitura às variáveis de estado da URL do Nerdlet. Por exemplo, um `entityGuid` no [explorador de entidade](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/).
* `nerdlet`: permite que você configure seu Nerdlet e grave no estado da URL do seu Nerdlet.
