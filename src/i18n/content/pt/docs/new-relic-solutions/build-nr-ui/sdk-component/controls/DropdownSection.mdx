Seção do componente `<Dropdown>` .

### Uso

```js
import { DropdownSection } from 'nr1'
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `children` Nó <h5>REQUERIDO</h5> <h5>| função</h5>
      </td>

      <td>
        Este componente pode ser renderizado de forma declarativa, passando diretamente um conjunto de filhos, ou virtualizado, passando um render retorno de chamada (função como filhos). Os únicos itens permitidos dentro (ou retornados pelo render retorno de chamada) são do tipo `<DropdownItem>`.A recomendação é utilizar o render retorno de chamada quando for fornecido um grande número de itens, pois a lista de itens será virtualizada pelo componente, aumentando assim o desempenho.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>corda</h5>
      </td>

      <td>
        Acrescenta nomes de classes ao componente. Deve ser usado apenas para fins de posicionamento e espaçamento.
      </td>
    </tr>

    <tr>
      <td>
        `items` <h5>matriz</h5>
      </td>

      <td>
        Itens a serem renderizados, no formato de uma lista de objetos. Normalmente, cada item na matriz de itens contém os dados necessários para gerar o `<DropdownItem>` correspondente. Esta propriedade é necessária ao renderizar itens com o retorno de chamada de renderização (funcionam como filhos).
      </td>
    </tr>

    <tr>
      <td>
        `onLoadMore` <h5>função</h5>
      </td>

      <td>
        retorno de chamada disparado quando mais itens devem ser carregados. Isso acontece quando você está carregando lentamente os itens e os itens que estão prestes a renderizar não podem ser encontrados na matriz `items`. Este retorno de chamada deve ser usado para buscar/carregar os itens ausentes do backend ou de outras fontes. A promessa retornada deve ser resolvida assim que os dados do item terminarem de carregar. Será usado para determinar quando atualizar a lista com os dados recém-carregados. Esse retorno de chamada pode ser chamado diversas vezes em reação a um único evento de rolagem.

        <FunctionDefinition
          returnValue={[]}
          arguments={[{"name":"cursor","type":"Cursor","description":"Items to load."}]}
        />
      </td>
    </tr>

    <tr>
      <td>
        `rowCount` <h5>número</h5>
      </td>

      <td>
        Número de linhas. Por padrão, é igual ao comprimento da matriz passada na propriedade items. Você deve especificar `rowCount` quando souber o número total de itens, mas quiser carregá-los lentamente durante a rolagem.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objeto</h5>
      </td>

      <td>
        Estilo embutido para estilo personalizado. Deve ser usado apenas para fins de posicionamento e espaçamento.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>corda</h5>
      </td>

      <td>
        Adiciona um atributo `data-test-id` . Use-o para destino do componente em testes unitários e E2E. Para que um ID de teste seja válido, prefixe-o com seu ID nerdpack, seguido por um ponto. Por exemplo, `my-nerdpack.some-element`.**Nota:** Você pode não ver `data-test-id` atributo pois eles são removidos do DOM, para depurá-los passe um parâmetro de consulta `e2e-test` para a URL.
      </td>
    </tr>

    <tr>
      <td>
        `title` <h5>corda</h5>
      </td>

      <td>
        Título da seção.
      </td>
    </tr>
