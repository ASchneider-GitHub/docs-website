## Uso [#usage]

```js
import { Button } from 'nr1'
```

## Exemplos [#examples]

### Básico [#basic]

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>;
```

### Tamanhos [#sizes]

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.LARGE}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.MEDIUM}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.SMALL}>
    Click me
  </Button>
</div>;
```

```js
<div className="nr1-Docs-prettify">
```

      <td>
        `ariaControls`
      </td>

      <td>
        corda
      </td>

      <td>
        Passe a string `id` do elemento que o `Button` controla quando ele é usado para expandir ou abrir um painel. Use-o junto com `ariaExpanded`.

        **Botão expande um contêiner**

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
        <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
            Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
            Content
            </div>
        </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded`
      </td>

      <td>
        boleano
      </td>

      <td>
        Use-o para indicar se o elemento que o `Button` controla está expandido ou não.

        Recomendado verificar também os exemplos `ariaControls` e `ariaHasPopup` .
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup`
      </td>

      <td>
        boolean|string
      </td>

      <td>
        Use-o para indicar que `Button` alterna um elemento de sobreposição interativo. O valor pode ser:

        * `false` (padrão) indica que o elemento não possui um pop-up.

        * `true` indica que o pop-up tem uma função `menu` .

        * `"menu"` indica que o pop-up tem uma função `menu` .

        * `"listbox"` indica que o pop-up tem uma função `listbox` .

        * `"tree"` indica que o pop-up tem uma função `tree` .

        * `"grid"` indica que o pop-up tem uma função `grid` .

        * `"dialog"` indica que o pop-up tem uma função `dialog` . Leia [as especificações WAI ARIA](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup) para este atributo para saber qual delas se adapta melhor ao seu caso de uso.

          **Botão indicando que abre uma caixa de diálogo**

          ```js
          const { modalHidden } = this.state;

          return (
          <>
              <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
              >
              Open modal
              </Button>
              <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
              />
          </>
          );
          ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaLabel`
      </td>

      <td>
        corda
      </td>

      <td>
        Utilize-o para descrever melhor o contexto da ação do componente ou em botões que exibem apenas um ícone para o usuário em leitores de tela.

        **Botão apenas de ícone com melhor acessibilidade**

        ```js
        <Button
            iconType={Button.ICON_TYPE.INTERFACE__OPERATIONS__SEARCH}
            ariaLabel="Search for entities"
        />;
        ```
      </td>
    </tr>

      <td>
        `sizeType`
      </td>

      <td>
        enumeração
      </td>

      <td>
        Valor padrão: `Button.SIZE_TYPE.MEDIUM`. Recomendamos que você use o botão de tamanho normal em quase todas as instâncias.

        Use o botão fino com moderação, pois isso diminui a importância do botão. Não use-o apenas para caber em um espaço, mas considere aumentar o espaço ao redor de um botão padrão.

        Os poucos casos de uso de um botão grande são em material de marketing para seu complemento: introduzindo-o em uma página inicial ou em uma mensagem principal.

        ```
        <One of
            Button.SIZE_TYPE.LARGE,
            Button.SIZE_TYPE.MEDIUM,
            Button.SIZE_TYPE.SMALL,
        >
        ```
      </td>
    </tr>

        `spacingType`
      </td>

      <td>
        enumeração
      </td>

      <td>
        Propriedade de espaçamento. O espaçamento é definido como uma tupla de zero a quatro valores, que segue as mesmas convenções das propriedades CSS, como margem ou preenchimento. Para omitir um valor, use `SPACING_TYPE.OMIT`.

        ```
        <Array of
            <One of
                Button.SPACING_TYPE.EXTRA_LARGE,
                Button.SPACING_TYPE.LARGE,
                Button.SPACING_TYPE.MEDIUM,
                Button.SPACING_TYPE.NONE,
                Button.SPACING_TYPE.OMIT,
                Button.SPACING_TYPE.SMALL,
            >
        >
        ```
      </td>
    </tr>

        **Nota**: Você pode não ver `data-test-id` atributo pois eles são removidos do DOM, para depurá-los passe um parâmetro de consulta `e2e-test` para a URL.

    <tr>
      <td>
        tipo
      </td>

      <td>
        enumeração
      </td>

      <td>
        Valor padrão: `Button.TYPE.NORMAL`. O tipo pode ser:

        * Primário. Use para chamar a atenção para uma ação específica que você deseja que o usuário execute como próxima etapa.

        * Normal (usado como secundário) — use quando várias ações precisam ser exibidas.

        * Contorno. Use para ações importantes que não são a ação principal (primária) de uma determinada visualização. Essa variação fica entre os botões primário e padrão em termos de hierarquia.

        * Simples. Use quando várias ações precisam estar disponíveis e são menos importantes para o usuário.

        * Destrutivo. Use quando você tiver uma ação destrutiva, como excluir ou remover, que gostaria que o usuário pausasse e considerasse antes de concluir.

          ```
          <One of
              Button.TYPE.DESTRUCTIVE,
              Button.TYPE.NORMAL,
              Button.TYPE.OUTLINE,
              Button.TYPE.PLAIN,
              Button.TYPE.PLAIN_NEUTRAL,
              Button.TYPE.PRIMARY,
          >
          ```
      </td>
    </tr>
