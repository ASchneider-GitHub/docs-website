Envolve o conteúdo de uma célula da tabela localizada no cabeçalho da tabela, mantendo-os no topo e adicionando controles adicionais conforme necessário, por exemplo mecanismos de classificação.

### Uso

```js
import { TableHeaderCell } from 'nr1'
```

      <td>
        Método para classificar linhas. O método padrão geralmente fará a coisa certa (ordenação alfabética para texto, ordem numérica para números e agrupamento booleano), mas se necessário, isso pode ser substituído. A função leva dois parâmetros (`a` e `b`) e deve retornar um número negativo se `a` for considerado que aparece antes de `b`, um número positivo se for considerado que aparece depois e zero se ambos os valores forem idênticos. Cada um dos objetos passados para o retorno de chamada contém o valor para o dado linha (ou seja, o resultado da chamada de `value` sobre a linha), o item da linha (como `item`), o índice da linha (como `index`) e todos os itens na matriz `items` (como `items`). Geralmente você comparará linhas usando `a.item` e `b.item`.

        `value` <h5>função</h5>
      </td>

      <td>
        Função usada para extrair o valor bruto que representa uma coluna. O valor bruto pode ser um booleano (por exemplo, se você renderizar uma caixa de seleção), um número (se você renderizar uma métrica) ou texto para outros. Ele é chamado com um objeto contendo o item que representa a linha, seu índice no `items` matriz passada para a tabela, e os próprios itens. Esta função é usada para ordenar a tabela (pelo valor fornecido) bem como para a heurística de `'fit-content'` larguras.
      </td>

    <tr>
      <td>
        `width` <h5>corda</h5>
      </td>

      <td>
        Representa a largura da coluna. É sempre uma string e pode ser de um dos seguintes tipos:

        * `'fit-content'`: a coluna ocupará o espaço mínimo necessário para alocar todos os itens na coluna. Para listas virtualizadas, uma heurística será usada para evitar a renderização de todas as células.
        * `'YYpx'`: onde `YY` é um número, representa a largura da coluna em pixels.
        * `'YY%'`: onde `YY` é um número, representa a largura da coluna em porcentagem, relativa ao tamanho do espaço disponível onde a tabela está sendo renderizada.
        * `'YYfr'`: onde `YY` é um número, representa uma fração da largura disponível. As frações são proporcionais a outras; para instância, uma tabela com duas colunas (`1fr` e `2fr`) terá a segunda coluna duas vezes maior que a primeira. Se, após calcular as larguras, o tamanho geral da tabela não cobrir o espaço disponível, todas as colunas serão esticadas proporcionalmente para caber em todo o espaço disponível.
      </td>
