New Relic ofrece una integración con Atlassian Jira que habilita la pestaña de incidente característica dentro de Jira. Esta característica sincroniza toda la información de incidentes y problemas New Relic con Atlassian. Estos datos luego aparecen en Jira en la pestaña de incidentes y le permiten conectarlos con problemas de Jira para mejorar su flujo de trabajo de interrupción. Las secciones cubren cómo configurar esta integración y las mejores prácticas de uso.

<Callout title="PREVISUALIZAR característica">
  Esta característica aún está en versión preliminar, ¡pero te animamos a que la pruebes!
</Callout>

## Compartir datos [#data-sharing]

Una vez que instales la integración, permitirás que New Relic for Jira realice estas acciones:

* Lea entidad, incidente y problemas de Jira.
* Escribe entidad, incidente y issues a Jira.
* Elimine entidad, incidente y problemas tanto de New Relic como de Jira para reflejar los informes de datos con cada entidad.

<Callout variant="caution">
  Esta integración expondrá sus problemas de New Relic e información sobre incidentes a todos los usuarios en el espacio de trabajo de Jira relacionado. Es posible que el usuario de Jira en este espacio de trabajo no tenga acceso a New Relic e, incluso si lo tuviera, es posible que no tenga acceso a las mismas cuentas New Relic que su usuario de clave de API.

  Si tiene información de cuenta de New Relic para ocultar, utilice un [usuario de integración](/docs/more-integrations/best-practices-integration/#integration-user-leverage) que no tenga acceso a sus cuentas confidenciales.
</Callout>

## Instalacion [#installation]

Siga estos pasos para integrar New Relic con Atlassian Jira.

<Steps>
  <Step>
    ## Instale la aplicación del mercado New Relic [#install-marketplace]

    Debes instalar la New Relic Errors Inbox para Jira para mantener tus datos sincronizados entre New Relic y Atlassian. Tenga en cuenta que debe tener el permiso de administrador del sitio Jira para instalar la aplicación Marketplace. Vaya a [Atlassian Marketplace](https://marketplace.atlassian.com/apps/1227457/new-relic-errors-inbox-for-jira?hosting=cloud&tab=overview) y haga clic en el botón **Install** .

    <Callout variant="tip">
      Si no tiene el permiso de administrador del sitio, puede solicitarlo a su administrador. Le recomendamos que haga un seguimiento con el administrador de su sitio Jira para completar el proceso de instalación de la aplicación.
    </Callout>
  </Step>

  <Step>
    ## Crear una clave API de New Relic [#create-new-relic-key]

    Después de la instalación de la aplicación New Relic Atlassian Marketplace, debe crear una [clave API deNew Relic ](/docs/apis/intro-apis/new-relic-api-keys/). El administrador del sitio de Jira y el administrador de New Relic decidirán sobre las cuentas expuestas a Jira. El usuario que tenga el acceso correcto a la cuenta de New Relic , podrá generar una clave de API de New Relic . La clave de API de New Relic tiene acceso a las mismas cuentas que el usuario que las generó.

    Cree y administre su clave de API desde la [New Relic página de clave de API UI](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) o en la página log del [API token de](https://id.atlassian.com/manage-profile/security/api-tokens) Atlassian y haga clic en **Create API token**.
  </Step>

  <Step>
    ## Aplicar una clave de API de New Relic [#apply-new-relic-key]

    Una vez que haya creado una clave de API, puede aplicar esa clave a la primera página de la aplicación New Relic Connect.

    Pegue su token API de New Relic en la aplicación New Relic Atlassian Marketplace y haga clic en **Connect**.

    <img
      width="50%;"
      title="Apply a New Relic API key"
      alt="Apply a New Relic API key"
      src={opentelemetryAtlassianNRapiToken}
    />
  </Step>

  <Step>
    ## Seleccione sus servicios New Relic [#select-nr-services]

    Una vez que haya creado una clave de API en la aplicación New Relic Atlassian Marketplace, verá una lista de todas las entidades asociadas con las cuentas en esa clave de API. Abra la página de gestión de incidentes de Jira y seleccione todas las entidades que desea exponer en Jira.

    <img
      title="Jira incident management page"
      alt="Jira incident management page"
      src={opentelemetryJiraIncidentManagement}
    />
  </Step>

  <Step>
    ## Ver problemas e incidentes New Relic en Atlassian Jira [#see-issues-incidents]

    Después de haber seleccionado su entidad New Relic , la aplicación comienza a sincronizar datos de New Relic en Atlassian Jira. Abra la pestaña de incidentes para ver una lista de incidentes detectados.
  </Step>
</Steps>

## Habilitar la pestaña de incidentes [#enable-incidents-tab]

Es posible que deba vincularse a la documentación de Atlassian. Guía al administrador del sitio de Jira para habilitar la pestaña de incidentes y asignarla a un espacio de trabajo. Existe la posibilidad de que se habiliten de forma predeterminada y entonces no necesitarás esta sección.

## Eliminar una conexión entre New Relic y Atlassian Jira [#remove-connection]

Puede dejar de sincronizar datos de New Relic en cualquier momento y eliminar los datos sincronizados. Como administrador del sitio de Jira, puede hacerlo desde la página de configuración de Atlassian Marketplace. Puede seleccionar su organización conectada y hacer clic en **Delete**. Una vez finalizado, ya no verás información sobre incidentes ni problemas de New Relic en la pestaña de incidentes en Jira.

## Conectarse a una organización o usuario diferente [#different-conexion]

En cualquier momento, puede conectarse a la organización New Relic incorrecta o puede querer ajustar el acceso a la cuenta que proporciona su clave de API. Puedes hacerlo desde tu página de Atlassian Marketplace. Cuando haces clic en **Connect another New Relic organization**, deberás proporcionar una clave API de New Relic . Dar una nueva clave eliminará su conexión actual e incluirá todos los datos sincronizados con Atlassian. Luego, se iniciará un nuevo proceso de sincronización con el acceso de la cuenta y organización de la nueva clave de API.
