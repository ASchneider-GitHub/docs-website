Cuando un usuario elige una o más opciones proporcionadas

El estado de la casilla de verificación se puede controlar a través de las propiedades `checked` y `indeterminate` . Si no se proporciona ninguno de ellos, la casilla de verificación gestionará su propio estado.

Puede escuchar los cambios de estado de la casilla de verificación a través de la propiedad `onChange` .

**Nota**: La configuración `checked` anulará `defaultChecked` ya que coloca el componente en un estado controlado. `defaultChecked` solo debe usar para casillas de verificación no controladas.

### Uso

```js
import { Checkbox } from 'nr1'
```

### Ejemplos

#### Básico

```js
<div className="nr1-Docs-prettify">
  <Checkbox onChange={(event) => alert('Foo')} label="Foo" />
  <Checkbox indeterminate label="Bar" />
  <Checkbox checked disabled label="Baz" />
</div>
```

#### con informacion

```js
<Checkbox onChange={(event) => alert('Foo')} info="Info value" label="Foo" />
```

#### Con descripción

```js
<Checkbox
  onChange={(event) => alert('Foo')}
  description="Description value"
  label="Foo"
/>
```

#### Con mensaje inválido

```js
<Checkbox
  onChange={(event) => alert('Foo')}
  invalid="Invalid message value"
  label="Foo"
/>
```

#### Componente controlado

```js
class MyNerdlet extends React.PureComponent {
  constructor(props) {
    super(props);


    this.state = {
      isChecked: false,
    };


    this.onChange = this.onChange.bind(this);
  }


  onChange(event) {
    console.log(event);
    const isChecked = event.target.checked;


    this.setState({ isChecked });
  }


  render() {
    return (
      <Checkbox
        checked={this.state.isChecked}
        onChange={this.onChange}
        label="Foo"
      />
    );
  }
}
```
