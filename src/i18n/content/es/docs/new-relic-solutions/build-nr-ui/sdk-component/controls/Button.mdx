Los botones se emplean en todo el producto para proporcionar al usuario una forma de completar una acción. Trate de no abusar de los botones en su experiencia. Esto ayudará al usuario a saber exactamente qué acción le gustaría que realice. Nunca emplees más de un botón principal en tu experiencia.

Capacidades primarias: los botones pueden

* realizar una acción
* realizar múltiples acciones, si hay varios botones presentes.

### Uso

```js
import { Button } from 'nr1'
```

### Ejemplos

#### Básico

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={
Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={
Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `ariaControls` <h5>cadena</h5>
      </td>

      <td>
        Pase la cadena `id` del elemento que controla `Button` cuando se usa para expandir o abrir un panel. Úselo junto con `ariaExpanded`.

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
          <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
              Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
              Content
            </div>
          </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded` <h5>booleano</h5>
      </td>

      <td>
        Úselo para indicar si el elemento que controla `Button` está expandido o no. Se recomienda consultar también los ejemplos `ariaControls` y `ariaHasPopup` .
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup` <h5>boolean|string</h5>
      </td>

      <td>
        Úselo para indicar que `Button` alterna un elemento de superposición interactivo. El valor puede ser:

        * `false` (predeterminado) indica que el elemento no tiene una ventana emergente.
        * `true` indica que la ventana emergente tiene una función `menu`
        * `"menu"` indica que la ventana emergente tiene una función `menu`
        * `"listbox"` indica que la ventana emergente tiene una función `listbox`
        * `"tree"` indica que la ventana emergente tiene una función `tree`
        * `"grid"` indica que la ventana emergente tiene una función `grid`
        * `"dialog"` indica que la ventana emergente tiene una función `dialog` Lea [las especificaciones WAI ARIA para este atributo](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup) para saber cuál se adapta mejor a su caso de uso.

        ```js
        const { modalHidden } = this.state;

        return (
          <>
            <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
            >
              Open modal
            </Button>
            <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
            />
          </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaLabel` <h5>cadena</h5>
      </td>

      <td>
        Úselo para describir mejor el contexto de la acción del componente o en botones que muestran solo un ícono para el usuario en los lectores de pantalla.

        ```js
        <Button
          iconType={Button.ICON_TYPE.INTERFACE__OPERATIONS__SEARCH}
          ariaLabel="Search for entities"
        />;
        ```
      </td>
    </tr>

    <tr>
      <td>
        `children` <h5>nodo</h5>
      </td>

      <td>
        Contenido para renderizar dentro del botón.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>cadena</h5>
      </td>

      <td>
        Agrega nombres de clase al componente. Debe usar solo con fines de posicionamiento y espaciado.
      </td>
    </tr>

    <tr>
      <td>
        `disabled` <h5>booleano</h5>
      </td>

      <td>
        Emplee el estado deshabilitado para un botón antes de que un usuario complete los campos obligatorios de un formulario o cuando un usuario deba completar alguna otra tarea antes de que se pueda habilitar el botón.

      <td>
        El tipo puede ser:

        * Principal: empléelo para llamar la atención sobre una acción específica que desea que el usuario realice como siguiente paso.

        * Normal (usado como secundario): empléelo cuando sea necesario mostrar múltiples acciones.

        * Esquema: empléelo para acciones importantes que no son la acción principal (primaria) de una vista determinada. Esta variación se encuentra entre los botones principal y predeterminado en términos de jerarquía.

        * Sencillo: empléelo cuando sea necesario disponer de varias acciones que sean menos importantes para el usuario.

        * Destructivo: empléelo cuando tenga una acción destructiva como eliminar o eliminar, que le gustaría que el usuario pausara y considerara antes de completarla.
