## Uso [#usage]

```js
import { Button } from 'nr1'
```

## Ejemplos [#examples]

### Básico [#basic]

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>;
```

### Tallas [#sizes]

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.LARGE}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.MEDIUM}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.SMALL}>
    Click me
  </Button>
</div>;
```


      <td>
        `ariaControls`
      </td>

      <td>
        cadena
      </td>

      <td>
        Pase la cadena `id` del elemento que controla `Button` cuando se usa para expandir o abrir un panel. Úselo junto con `ariaExpanded`.

        **El botón expande un contenedor**

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
        <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
            Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
            Content
            </div>
        </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded`
      </td>

      <td>
        booleano
      </td>

      <td>
        Úselo para indicar si el elemento que controla `Button` está expandido o no.

        Se recomienda consultar también los ejemplos `ariaControls` y `ariaHasPopup` .
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup`
      </td>

      <td>
        boolean|string
      </td>

      <td>
        Úselo para indicar que `Button` alterna un elemento de superposición interactivo. El valor puede ser:

        * `false` (predeterminado) indica que el elemento no tiene una ventana emergente.

        * `true` indica que la ventana emergente tiene una función `menu` .

        * `"menu"` indica que la ventana emergente tiene una función `menu` .

        * `"listbox"` indica que la ventana emergente tiene una función `listbox` .

        * `"tree"` indica que la ventana emergente tiene una función `tree` .

        * `"grid"` indica que la ventana emergente tiene una función `grid` .

        * `"dialog"` indica que la ventana emergente tiene una función `dialog` . Lea [las especificaciones WAI ARIA](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup) de este atributo para saber cuál se adapta mejor a su caso de uso.

          **Botón que indica que abre un diálogo.**

          ```js
          const { modalHidden } = this.state;

          return (
          <>
              <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
              >
              Open modal
              </Button>
              <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
              />
          </>
          );
          ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaLabel`

        cadena

        ```js
        <Button

            ariaLabel="Search for entities"
        />;
        ```

      <td>
        `sizeType`
      </td>

      <td>
        enumeración
      </td>

      <td>
        Valor predeterminado: `Button.SIZE_TYPE.MEDIUM`. Le recomendamos emplear el botón de tamaño normal en casi todos los casos.

        Emplee el botón delgado con moderación, ya que disminuye la importancia del botón. No lo use únicamente para caber en un espacio, pero considere aumentar el espacio alrededor de un botón predeterminado.

        Los pocos casos en los que se emplea un botón grande se encuentran en material similar a marketing para su complemento: presentándolo en una página de inicio o en un mensaje de héroe.

        ```
        <One of
            Button.SIZE_TYPE.LARGE,
            Button.SIZE_TYPE.MEDIUM,
            Button.SIZE_TYPE.SMALL,
        >
        ```
      </td>
    </tr>

        ```
                Button.SPACING_TYPE.LARGE,
                Button.SPACING_TYPE.MEDIUM,
                Button.SPACING_TYPE.NONE,
                Button.SPACING_TYPE.OMIT,
                Button.SPACING_TYPE.SMALL,
            >
        >
        ```

    <tr>
      <td>
        tipo
      </td>

      <td>
        enumeración
      </td>

      <td>
        Valor predeterminado: `Button.TYPE.NORMAL`. El tipo puede ser:

        * Primario. Úselo para llamar la atención sobre una acción específica que desea que el usuario realice como siguiente paso.

        * Normal (usado como secundario): empléelo cuando sea necesario mostrar múltiples acciones.

        * Describir. Úselo para acciones importantes que no son la acción principal (primaria) de una vista determinada. Esta variación se encuentra entre los botones principal y predeterminado en términos de jerarquía.

        * Plano. Empléelo cuando sea necesario disponer de varias acciones que sean menos importantes para el usuario.

        * Destructivo. Úselo cuando tenga una acción destructiva como eliminar o eliminar, que le gustaría que el usuario pausara y considerara antes de completarla.

          ```
          <One of
              Button.TYPE.DESTRUCTIVE,
              Button.TYPE.NORMAL,
              Button.TYPE.OUTLINE,
              Button.TYPE.PLAIN,
              Button.TYPE.PLAIN_NEUTRAL,
              Button.TYPE.PRIMARY,
          >
          ```
      </td>
    </tr>
