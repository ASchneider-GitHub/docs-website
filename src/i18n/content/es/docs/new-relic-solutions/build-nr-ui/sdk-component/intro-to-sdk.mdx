Para ayudarlo a crear la aplicación New Relic , le proporcionamos los componentes del SDK New Relic , que se emplean para crear aplicaciones New Relic .

Aquí le brindamos una introducción a los tipos de API de llamada y componentes en el SDK. El SDK proporciona todo lo que necesita para crear sus Nerdlets, crear visualizaciones y obtener datos de New Relic o de terceros.

## Componentes del SDK [#components]

Puede encontrar los componentes del SDK en el paquete del módulo Node denominado `nr1`, que obtiene cuando instala la [CLI de New Relic](/docs/new-relic-solutions/tutorials/new-relic-cli/#install-the-new-relic-cli). Los `nr1` componentes se dividen en varias categorías:

* Componentes UI
* Componentes del gráfico
* Componentes de consulta y almacenamiento.
* API de plataforma

### Componentes UI [#ui-components]

La categoría **UI components** del SDK contiene componentes de React UI, que incluyen:

* **Componentes de texto**: estos componentes proporcionan elementos básicos de fuente y encabezado. Estos incluyen `HeadingText` y `BlockText`.

* **Componentes de diseño**: estos componentes le brindan control sobre el diseño y lo ayudan a crear diseños de diseño complejos sin tener que lidiar con CSS. Los componentes de diseño incluyen:

  * `Grid` y `GridItem`: para organizar contenido de página más complejo y de mayor escala en filas y columnas.
  * `Stack` y `StackItem`: para organizar contenido de página más simple y de menor escala, en columnas o filas.
  * `Tabs` y `TabsItem`: agrupa varios contenidos relacionados en secciones independientes que se pueden ocultar.
  * `List` y `ListItem`: para proporcionar un esqueleto básico de listas virtualizadas.
  * `Card`, `CardHeader` y `CardBody`: se emplean para agrupar conceptos y tareas similares.

* **Componentes del formulario**: estos componentes proporcionan los componentes básicos para interactuar con la UI. Estos incluyen [`Button`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/button), [`TextField`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/text-field), [`Dropdown`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown), [`DropdownItem`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown-item), [`Checkbox`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/checkbox), [`RadioGroup`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio-group) y [`Radio`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio).

* **Componentes de retroalimentación**: emplee estos componentes para proporcionar retroalimentación al usuario sobre las acciones que realizó. Estos incluyen: `Spinner` y `Toast`.

* **Componentes superpuestos**: emplee estos componentes para mostrar información contextual y opciones en forma de una vista secundaria adicional que aparece encima de otro contenido en la pantalla cuando se activa una acción o evento. Pueden requerir la interacción del usuario (como modales) o aumentar (como una información sobre herramientas). Estos incluyen: `Modal` y `Tooltip`.

<Callout variant="important">
  Los componentes con el sufijo `Item` solo pueden funcionar como hijos directos de ese nombre sin el sufijo. Por ejemplo: `GridItem` solo debe encontrar como hijo de `Grid`.
</Callout>

### Componentes del gráfico [#chart-components]

La categoría **Charts** del SDK contiene componentes que representan diferentes tipos de gráficos. El componente `ChartGroup` ayuda a un grupo de gráficos relacionados a compartir datos y estar alineados.

Algunos componentes del gráfico pueden realizar [consultasNRQL ](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/)por sí solos; algunos aceptan un conjunto de datos personalizado.

### Componentes de consulta y almacenamiento. [#query-storage]

La categoría **Query components** contiene componentes para buscar y almacenar datos de New Relic.

La forma principal de recuperar datos es con NerdGraph, nuestro extremo GraphQL. Esto se puede consultar usando `NerdGraphQuery`. Para simplificar el uso de NerdGraph consulta, proporcionamos algunos componentes con consulta predefinida. Consulte nuestros documentos sobre [NerdGraph](/collect-data/get-started-nerdgraph-api-explorer) para obtener más información.

También proporcionamos almacenamiento para almacenar un pequeño conjunto de datos, como datos de configuración o datos específicos del usuario. Para obtener más información sobre esto, consulte [NerdStorage](/docs/new-relic-solutions/build-nr-ui/nerdstorage).

### API de plataforma

Los componentes de la plataforma API del SDK permiten que su aplicación interactúe con diferentes partes de la plataforma New Relic , leyendo y escribiendo el estado desde y hacia la URL, estableciendo la configuración, etc. Se dividen en estas categorías:

* `PlatformStateContext`: proporciona acceso de lectura a las variables de estado de la URL de la plataforma. Por ejemplo `timeRange` en el [selector de tiempo](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker).
* `navigation`: Un objeto que permite la manipulación programática de la navegación en New Relic. Ejemplo: abrir un nuevo Nerdlet.
* `NerdletStateContext`: Proporciona acceso de lectura a las variables de estado de la URL de Nerdlet. Por ejemplo un `entityGuid` en el [explorador de entidades](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/).
* `nerdlet`: le permite configurar su Nerdlet y escribir en el estado URL de su Nerdlet.
