Sección del componente `<Dropdown>` .

### Uso

```js
import { DropdownSection } from 'nr1'
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `children` Nodo <h5>REQUERIDO</h5> <h5>|función</h5>
      </td>

      <td>
        Este componente se puede representar de forma declarativa, pasando directamente un conjunto de elementos secundarios o virtualizado, pasando una devolución de llamada de procesamiento (funciona como elementos secundarios). Los únicos elementos permitidos dentro (o devueltos por la devolución de llamada de procesamiento) son del tipo `<DropdownItem>`. La recomendación es emplear la devolución de llamada de renderizado cuando se proporciona una gran cantidad de elementos, ya que el componente virtualizará la lista de elementos, lo que aumentará el rendimiento.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>cadena</h5>
      </td>

      <td>
        Agrega nombres de clase al componente. Debe usar solo con fines de posicionamiento y espaciado.
      </td>
    </tr>

    <tr>
      <td>
        `items` <h5>matriz</h5>
      </td>

      <td>
        Elementos a renderizar, en forma de lista de objetos. Por lo general, cada elemento de la matriz de elementos contiene los datos necesarios para generar el `<DropdownItem>` correspondiente. Este accesorio es necesario al representar elementos con la devolución de llamada de renderizado (función como elementos secundarios).
      </td>
    </tr>

    <tr>
      <td>
        `onLoadMore` <h5>función</h5>
      </td>

      <td>
        La devolución de llamada se activa cuando se deben cargar más artículos. Esto sucede cuando carga los elementos de forma diferida y los elementos que están a punto de renderizarse no se pueden encontrar en la matriz `items` . Esta devolución de llamada debe usar para recuperar/cargar los elementos que faltan desde el backend u otras fuentes. La promesa devuelta debe resolver una vez que los datos del elemento terminaron de cargar. Se empleará para determinar cuándo actualizar la lista con los datos recién cargados. Esta devolución de llamada se puede llamar varias veces como reacción a un único evento de desplazamiento.

        <FunctionDefinition
          returnValue={[]}
          arguments={[{"name":"cursor","type":"Cursor","description":"Items to load."}]}
        />
      </td>
    </tr>

    <tr>
      <td>
        `rowCount` <h5>número</h5>
      </td>

      <td>
        Número de filas. De forma predeterminada, es igual a la longitud de la matriz pasada en la propiedad de elementos. Debe especificar `rowCount` cuando conozca el número total de elementos pero desee cargarlos de forma diferida mientras se desplaza.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objeto</h5>
      </td>

      <td>
        Estilo en línea para un estilo personalizado. Debe usar solo con fines de posicionamiento y espaciado.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>cadena</h5>
      </td>

      <td>
        Agrega un atributo `data-test-id` . Úselo para apuntar al componente en pruebas unitarias y E2E. Para que una identificación de prueba sea válida, anteponga su identificación de nerdpack, seguido de un punto. Por ejemplo, `my-nerdpack.some-element`.**Nota:** Es posible que no vea el atributo `data-test-id` ya que se eliminan del DOM. Para depurarlos, pase un parámetro de consulta `e2e-test` a la URL.
      </td>
    </tr>

    <tr>
      <td>
        `title` <h5>cadena</h5>
      </td>

      <td>
        Sección de título.
      </td>
    </tr>
  </tbody>
</table>
