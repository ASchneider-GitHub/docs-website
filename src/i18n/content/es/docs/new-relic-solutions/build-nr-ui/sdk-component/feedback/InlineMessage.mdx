Emplee los mensajes en línea cuando un elemento requiera atención. Alertan al usuario de que hay información importante disponible o de que se requiere acción y desaparecen una vez que el usuario tomó la acción adecuada.

El mensaje en línea emplea un ícono como elemento principal para brindar un indicador visual del tipo y urgencia del mensaje. El icono se puede combinar con una etiqueta opcional para resaltar su mensaje.

El mensaje en línea revela un cuadro de diálogo que proporciona un resumen del problema y enlaces a información o acciones cuando el usuario pasa el cursor sobre el ícono o la combinación de ícono y etiqueta.

### Uso

```js
import { InlineMessage } from 'nr1'
```

### Ejemplos

#### Tipos

```js
<Stack>
  <StackItem>
    <InlineMessage label="We have updated the data for this chart." />
  </StackItem>
  <StackItem>
    <InlineMessage
      type={InlineMessage.TYPE.SUCCESS}
      label="The entity has been successfully created"
    />
  </StackItem>
  <StackItem>
    <InlineMessage
      type={InlineMessage.TYPE.WARNING}
      label="The edition is restricted for this account."
    />
  </StackItem>
  <StackItem>
    <InlineMessage
      type={InlineMessage.TYPE.CRITICAL}
      label="We couldn't fetch the data."
    />
  </StackItem>
</Stack>
```

#### Etiqueta y acción

```js
<InlineMessage
  type={InlineMessage.TYPE.CRITICAL}
  label="We couldn't transform the query."
  action={{ label: 'See the documentation', onClick: console.log }}
/>
```

#### Diálogo con detalles

```js
<InlineMessage
  type={InlineMessage.TYPE.SUCCESS}
  label="The data has been added."
  description="You have successfully added 3 new hosts."
  action={{ label: 'See the hosts', onClick: console.log }}
/>
```

#### Solo icono

```js
<InlineMessage
  type={InlineMessage.TYPE.CRITICAL}
  title="We can't render this query."
  description="We tried to transform your query into a query that is compatible with both Metric and previous Infrastructure methods, but some expressions weren’t available."
  action={{ label: 'See NRQL documentation', to: '//docs.newrelic.com' }}
/>
```
