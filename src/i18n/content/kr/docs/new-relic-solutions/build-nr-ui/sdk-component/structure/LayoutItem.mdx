`<Layout>` 구성요소의 하위 요소입니다.

기본적으로 레이아웃 영역의 전체 너비에 걸쳐 기본 유형을 사용합니다. 왼쪽 분할 및 오른쪽 분할은 고정된 왼쪽/오른쪽 영역을 제공합니다. 레이아웃은 각각 한 가지 유형만 지원합니다.

### 용법

```js
import { LayoutItem } from 'nr1'
```

### 소품

  <tbody>

      <td>

      </td>

      <td>
        하위 항목 내부에 표시할 콘텐츠입니다.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>끈</h5>
      </td>

      <td>
        구성 요소에 클래스 이름을 추가합니다.
      </td>
    </tr>

    <tr>
      <td>
        `sizeType` <h5>열거</h5>
      </td>

      <td>
        레이아웃 항목의 너비를 결정합니다. 탐색 목적으로는 작은 크기를 사용하는 것이 좋습니다(예: 사이드바) 및 상황에 맞는 정보를 표시하기 위한 중간 크기(예: 활동 스트림). 크기 유형은 SPLIT_LEFT 및 SPLIT_RIGHT 항목에서만 작동합니다.

        <OptionReference>
          LayoutItem.SIZE_TYPE.MEDIUM, LayoutItem.SIZE_TYPE.SMALL,
        </OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>물체</h5>
      </td>

      <td>
        사용자 정의 스타일을 위한 인라인 스타일.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>끈</h5>
      </td>

      <td>
        `data-test-id` 속성을 추가합니다. 이를 사용하여 단위 및 E2E 테스트의 구성 요소를 목표로 삼습니다. 테스트 ID가 유효하려면 앞에 nerdpack ID를 붙이고 그 뒤에 점이 옵니다. 예를 들어 `my-nerdpack.some-element`.

        **참고:** `data-test-id` 속성이 DOM 에서 제거될 때 표시되지 않을 수 있습니다. 이를 디버그하려면 `e2e-test` 쿼리 변수를 URL에 전달해야 합니다.
      </td>
    </tr>

    <tr>
      <td>
        `type` <h5>열거</h5>
      </td>

      <td>
        유형은 다음과 같습니다.

        * Main — 각 보기에 기본 콘텐츠를 배치하는 데 사용됩니다. 이는 레이아웃 영역의 전체 너비에 걸쳐 있습니다.

        * 왼쪽 분할 — 레이아웃 왼쪽의 고정 영역으로 렌더링됩니다.

        * 오른쪽 분할 — 레이아웃 오른쪽의 고정 영역으로 렌더링됩니다.

          <OptionReference>
            LayoutItem.TYPE.MAIN, LayoutItem.TYPE.SPLIT_LEFT, LayoutItem.TYPE.SPLIT_RIGHT,
          </OptionReference>
      </td>
    </tr>
