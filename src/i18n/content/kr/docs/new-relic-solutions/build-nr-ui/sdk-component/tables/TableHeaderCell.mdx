테이블 헤더에 있는 테이블 셀의 내용을 상단에 유지하고 필요에 따라 추가 컨트롤을 추가하여 래핑합니다. 정렬 메커니즘.

### 용법

```js
import { TableHeaderCell } from 'nr1'
```

### 소품

      </td>

      <td>
        셀 내부의 텍스트 정렬을 정의합니다.

          TableHeaderCell.ALIGNMENT_TYPE.CENTER, TableHeaderCell.ALIGNMENT_TYPE.LEFT, TableHeaderCell.ALIGNMENT_TYPE.RIGHT,
        </OptionReference>
      </td>

    <tr>
      <td>
        `children` <h5>마디</h5>
      </td>

      <td>
        테이블 셀의 내용입니다. 비어 있는 경우 사용자가 열에 표시되는 데이터에 대한 참조를 얻을 수 있도록 `title` 전달해야 합니다.
      </td>
    </tr>

      <td>
        `className` <h5>끈</h5>
      </td>

      <td>
        구성 요소에 클래스 이름을 추가합니다.
      </td>
    </tr>

    <tr>
      <td>
        `onClick` <h5>기능</h5>
      </td>

      <td>
        사용자가 헤더를 클릭할 때마다 콜백이 실행됩니다. 클릭하면 정렬 정보(및 다음 정렬 상태)가 두 번째 매개변수로 전달됩니다. 직접 계산하기보다는 이를 사용하십시오.

        <FunctionDefinition
          returnValue={[]}
          arguments={[{"name":"event","type":"React.MouseEvent","description":""},{"name":"sortingData","type":"Object","description":""}]}
        />
      </td>
    </tr>

    <tr>
      <td>
        `sortable` <h5>부울</h5>
      </td>

      <td>
        열이 정렬 가능한지 여부입니다. 즉, 제목 텍스트 옆에 위쪽 및 아래쪽 화살표가 표시됩니다. 정렬은 제어되는 구성 요소 인터페이스입니다. 정렬 이벤트를 수신하고 `sortingOrder` 를 변경하여 해당 열에 정렬을 적용하는 일은 사용자가 담당합니다. 소품.
      </td>
    </tr>

    <tr>
      <td>
        `sortingFunction` <h5>기능</h5>
      </td>

      <td>
        행 정렬 방법. 기본 방법은 일반적으로 올바른 작업(텍스트의 알파벳 순서, 숫자의 숫자 순서 및 부울 그룹화)을 수행하지만 필요한 경우 이를 재정의할 수 있습니다. 이 함수는 두 가지 매개변수(`a` 및 `b`)를 사용하며 `a` `b` 앞에 나타나는 것으로 간주되면 음수를 반환하고, 이후에 나타나는 것으로 간주되면 양수를, 두 값이 동일하면 0을 반환합니다. 콜백에 전달된 각 개체에는 주어진 값에 대한 값이 포함됩니다. 행(즉, 행에 대해 `value` 호출한 결과), 행 항목( `item`), 행의 인덱스( `index`) 및 `items` 에 있는 모든 항목( `items`). 일반적으로 `a.item` 및 `b.item` 사용하여 행을 비교합니다.

        <FunctionDefinition
          returnValue={[{"type":"number","description":""}]}
          arguments={[{"name":"a","type":"object","description":""},{"name":"b","type":"object","description":""}]}
        />
      </td>
    </tr>

    <tr>
      <td>
        `sortingOrder` <h5>숫자</h5>
      </td>

      <td>
        정렬 순서를 설정합니다. 가장 낮은 숫자는 가장 높은 정렬 우선순위를 나타냅니다. 즉, `0` 정렬된 첫 번째 열이 되고, 두 항목이 동일하면 `1` 열을 사용하게 됩니다. 명시적인 정렬 순서가 없는 열은 정렬 순서가 있는 모든 열 다음에 사용되며 표시되는 순서대로 적용됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `sortingType` <h5>열거</h5>
      </td>

      <td>
        열의 정렬을 설정합니다. 열은 `value` 소품이 반환한 요소를 기준으로 정렬됩니다.

        <OptionReference>
          TableHeaderCell.SORTING_TYPE.ASCENDING, TableHeaderCell.SORTING_TYPE.DESCENDING, TableHeaderCell.SORTING_TYPE.NONE,
        </OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>물체</h5>
      </td>

      <td>
        사용자 정의 스타일을 위한 인라인 스타일.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>끈</h5>
      </td>

      <td>
        `data-test-id` 속성을 추가합니다. 이를 사용하여 단위 및 E2E 테스트의 구성 요소를 목표로 삼습니다. 테스트 ID가 유효하려면 앞에 nerdpack ID를 붙이고 그 뒤에 점이 옵니다. 예를 들어 `my-nerdpack.some-element`.

        **참고:** `data-test-id` 속성이 DOM 에서 제거될 때 표시되지 않을 수 있습니다. 이를 디버그하려면 `e2e-test` 쿼리 변수를 URL에 전달해야 합니다.
      </td>
    </tr>

    <tr>
      <td>
        `title` <h5>끈</h5>
      </td>

      <td>
        셀의 제목을 설정합니다. 이는 열의 너비가 전체 제목을 표시할 수 없을 만큼 작을 때 도구 설명 목적으로 사용됩니다. 제공된 하위 항목의 문자열화된 버전이 기본값입니다.
      </td>
    </tr>

    <tr>
      <td>
        `value` <h5>기능</h5>
      </td>

      <td>
        열을 나타내는 원시 값을 추출하는 데 사용되는 함수입니다. 원시 값은 부울(예: 체크박스를 렌더링하는 경우), 숫자(지수를 렌더링하는 경우) 또는 기타 텍스트일 수 있습니다. 이는 행을 나타내는 항목을 포함하는 객체, `items` 의 색인과 함께 호출됩니다. 테이블과 항목 자체에 전달되었습니다. 이 함수는 테이블을 정렬(주어진 값 기준) 및 `'fit-content'` 너비의 경험적 방법에 사용됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `width` <h5>끈</h5>
      </td>

      <td>
        열의 너비를 나타냅니다. 이는 항상 문자열이며 다음 유형 중 하나일 수 있습니다.

        * `'fit-content'`: 열은 열의 모든 항목을 할당하는 데 필요한 최소 공간을 차지합니다. 가상화된 목록의 경우 모든 셀이 렌더링되지 않도록 경험적 방법이 사용됩니다.
        * `'YYpx'`: 여기서 `YY` 은 숫자이며 열 너비를 픽셀 단위로 나타냅니다.
        * `'YY%'`: 여기서 `YY` 은 숫자이며, 테이블이 렌더링되는 사용 가능한 공간의 크기를 기준으로 열의 너비를 백분율로 나타냅니다.
        * `'YYfr'`: 여기서 `YY` 은 숫자이며 사용 가능한 너비의 일부를 나타냅니다. 분수는 다른 분수에 비례합니다. 출력의 경우 두 개의 열(`1fr` 및 `2fr`)이 있는 테이블의 두 번째 열은 첫 번째 열의 두 배입니다. 너비를 계산한 후 테이블의 전체 크기가 사용 가능한 공간을 차지하지 않는 경우, 모든 열은 사용 가능한 모든 공간에 맞게 비례적으로 늘어납니다.
      </td>
    </tr>
  </tbody>
</table>
