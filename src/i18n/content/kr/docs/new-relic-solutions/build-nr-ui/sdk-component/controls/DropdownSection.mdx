`<Dropdown>` 구성요소의 섹션입니다.

### 용법

```js
import { DropdownSection } from 'nr1'
```

### 소품

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>필수</h5> <h5>노드|함수</h5>
      </td>

      <td>
        이 구성 요소는 자식 집합을 직접 전달하여 선언적으로 렌더링하거나 렌더링 콜백(자식으로 함수)을 전달하여 가상화할 수 있습니다. 내부에서 허용되는(또는 렌더링 콜백에 의해 반환되는) 유일한 항목은 `<DropdownItem>` 유형입니다. 권장 사항 항목 수가 많은 경우 렌더링 콜백을 사용하는 것입니다. 항목 목록이 구성 요소에 의해 가상화되어 성능이 향상되기 때문입니다.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>끈</h5>
      </td>

      <td>
        구성 요소에 클래스 이름을 추가합니다. 위치 지정 및 간격 지정 목적으로만 사용해야 합니다.
      </td>
    </tr>

    <tr>
      <td>
        `items` <h5>정렬</h5>
      </td>

      <td>
        객체 목록 형태로 렌더링할 항목입니다. 일반적으로 항목의 각 항목에는 해당 `<DropdownItem>` 생성하는 데 필요한 데이터가 포함되어 있습니다. 이 소품은 렌더링 콜백(하위 기능)을 사용하여 항목을 렌더링할 때 필요합니다.
      </td>
    </tr>

    <tr>
      <td>
        `onLoadMore` <h5>기능</h5>
      </td>

      <td>
        더 많은 항목을 로드해야 할 때 콜백이 실행됩니다. 이는 항목을 늦게 로드하여 렌더링하려는 항목을 `items` 에서 찾을 수 없을 때 발생합니다. 이 콜백은 백앤드 또는 다른 소스에서 누락된 항목을 가져오거나 로드하는 데 사용해야 합니다.반환된 항목 항목 데이터 로드가 완료되면 Promise가 해결되어야 합니다. 새로 로드된 데이터로 목록을 새로 고칠 시기를 결정하는 데 사용됩니다. 이 콜백은 단일 스크롤 이벤트에 대한 반응으로 여러 번 호출될 수 있습니다.

        <FunctionDefinition
          returnValue={[]}
          arguments={[{"name":"cursor","type":"Cursor","description":"Items to load."}]}
        />
      </td>
    </tr>

    <tr>
      <td>
        `rowCount` <h5>숫자</h5>
      </td>

      <td>
        행 수. 기본적으로 항목 소품에 전달된 다시의 길이와 같습니다. 총 항목 수를 알고 있지만 스크롤하는 동안 항목을 지연 로드하려는 경우 `rowCount` 를 지정해야 합니다.
      </td>
    </tr>

    <tr>
      <td>

      </td>

      <td>
        사용자 정의 스타일을 위한 인라인 스타일입니다. 위치 지정 및 간격 지정 목적으로만 사용해야 합니다.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>끈</h5>
      </td>

      <td>
        `data-test-id` 속성을 추가합니다. 이를 사용하여 단위 및 E2E 테스트의 구성 요소를 목표로 삼습니다. 테스트 ID가 유효하려면 앞에 nerdpack ID를 붙이고 그 뒤에 점이 옵니다. 예를 들어 `my-nerdpack.some-element`입니다.**참고:** `data-test-id` 속성이 DOM 에서 제거될 때 표시되지 않을 수 있습니다. 이를 디버그하려면 `e2e-test` 쿼리 변수를 URL에 전달해야 합니다.
