엔터티 범위 데이터에 대해 NerdStorage를 쿼리합니다.

전체 컬렉션 또는 단일 문서를 검색합니다.

### 용법

```js
import { EntityStorageQuery } from 'nr1'
```

### 예

#### 쿼리 수집

```js
<EntityStorageQuery
  entityGuid="MTIzNDU2fEZPT3xCQVJ8OTg3NjU0Mzcz"
  collection="foo"
>
  {({ loading, error, data }) => {
    if (loading) {
      return <Spinner />;
    }


    if (error) {
      return 'Error!';
    }


    return <pre>{JSON.stringify(data, null, 4)}</pre>;
  }}
</EntityStorageQuery>
```

#### 쿼리 문서

```js
<EntityStorageQuery
  entityGuid="MTIzNDU2fEZPT3xCQVJ8OTg3NjU0Mzcz"
  collection="foo"
  documentId="bar"
>
  {({ loading, error, data }) => {
    if (loading) {
      return <Spinner />;
    }


    if (error) {
      return 'Error!';
    }


    return <pre>{JSON.stringify(data, null, 4)}</pre>;
  }}
</EntityStorageQuery>
```

#### 필수 쿼리

```js
EntityStorageQuery.query({
  entityGuid: 'MTIzNDU2fEZPT3xCQVJ8OTg3NjU0Mzcz',
  collection: 'myCollection',
  documentId: 'myDocumentId',
}).then(({ data }) => console.log(data));
```

      <td>
        `fetchPolicyType` <h5>열거</h5>
      </td>

      <td>
        쿼리가 캐시된 데이터와 상호 작용하는 방법을 지정할 수 있습니다.

        * `CACHE_AND_NETWORK`: 쿼리는 가능한 경우 캐시에서 초기 데이터를 반환합니다. 그러나 전체 데이터가 캐시에 있는지 여부에 관계없이 쿼리는 항상 네트워크 인터페이스를 사용하여 요청하고 업데이트된 데이터를 반환합니다. 구성요소의 정적 `query()` 메소드를 사용하는 경우에는 이 옵션을 사용할 수 없습니다.

        * `CACHE_FIRST`: 쿼리는 쿼리에 대한 데이터가 아직 캐시에 없는 경우 **에만** 네트워크 인터페이스를 사용하여 요청합니다.

        * `CACHE_ONLY`: 쿼리는 네트워크 인터페이스를 사용하여 요청 **하지 않습니다** . 대신 캐시에서 사용 가능한 데이터를 반환합니다. 쿼리에 대한 데이터가 캐시에 없으면 오류가 발생합니다.

        * `NETWORK_ONLY`: 쿼리는 캐시에서 초기 데이터를 반환 **하지 않습니다** . 대신 **항상** 네트워크 인터페이스를 사용하여 요청합니다.

        * `NO_CACHE`: 쿼리는 캐시에서 초기 데이터를 반환 **하지 않습니다** . 대신 **항상** 네트워크 인터페이스를 사용하여 요청합니다. `NETWORK_ONLY` 정책과 달리 쿼리가 완료된 후 캐시에 데이터를 쓰지 않습니다.

          <OptionReference>
            EntityStorageQuery.FETCH_POLICY_TYPE.CACHE_AND_NETWORK, EntityStorageQuery.FETCH_POLICY_TYPE.CACHE_FIRST, EntityStorageQuery.FETCH_POLICY_TYPE.CACHE_ONLY, EntityStorageQuery.FETCH_POLICY_TYPE.NETWORK_ONLY, EntityStorageQuery.FETCH_POLICY_TYPE.NO_CACHE,
          </OptionReference>

        데이터의 범위를 행위자별로 지정해야 하는지 여부를 설정합니다. 즉, 읽거나 저장한 데이터가 객체에 접근하는 모든 사람에게 공통적인지, 아니면 사용자에게만 국한되는지 여부입니다. 객체(엔터티 또는 계정)는 "글로벌" 데이터와 "범위" 데이터를 모두 가질 수 있으며, 데이터는 읽기는 플래그의 상태에 따라 달라집니다.
