뉴렐릭 One SDK는 ApolloClient의 쿼리 구성 요소를 기반으로 쿼리 구성 요소를 제공합니다. 이러한 구성 요소는 Apollo Client 구성에 대한 걱정 없이 그리고 가장 일반적인 사용 사례의 경우 GraphQL 쿼리를 작성할 필요 없이 NerdGraph를 더 쉽게 쿼리할 수 있게 해주는 추상화 계층입니다.

NerdGraph의 모든 항목을 쿼리할 수 있는 일반 NerdGraph 쿼리 구성 요소입니다.

### 용법

```js
import { NerdGraphQuery } from 'nr1'
```

### 예

#### 실시예 1

```js
function render() {
  const query = `
        query($id: Int!) {
            actor {
                account(id: $id) {
                    name
                }
            }
        }
    `;


  const variables = {
    id: 1,
  };


  return (
    <NerdGraphQuery query={query} variables={variables}>
      {({ loading, error, data }) => {
        if (loading) {
          return <Spinner />;
        }


        if (error) {
          return 'Error!';
        }


        return <BlockText>{data.actor.account.name}</BlockText>;
      }}
    </NerdGraphQuery>
  );
}
```

        `fetchPolicyType` <h5>열거</h5>
      </td>

      <td>
        쿼리가 캐시된 데이터와 상호 작용하는 방법을 지정할 수 있습니다.

        * `CACHE_AND_NETWORK`: 쿼리는 가능한 경우 캐시에서 초기 데이터를 반환합니다. 그러나 전체 데이터가 캐시에 있는지 여부에 관계없이 쿼리는 항상 네트워크 인터페이스를 사용하여 요청하고 업데이트된 데이터를 반환합니다. 구성요소의 정적 `query()` 메소드를 사용하는 경우에는 이 옵션을 사용할 수 없습니다.

        * `CACHE_FIRST`: 쿼리는 쿼리에 대한 데이터가 아직 캐시에 없는 경우 **에만** 네트워크 인터페이스를 사용하여 요청합니다.

        * `CACHE_ONLY`: 쿼리는 네트워크 인터페이스를 사용하여 요청 **하지 않습니다** . 대신 캐시에서 사용 가능한 데이터를 반환합니다. 쿼리에 대한 데이터가 캐시에 없으면 오류가 발생합니다.

        * `NETWORK_ONLY`: 쿼리는 캐시에서 초기 데이터를 반환 **하지 않습니다** . 대신 **항상** 네트워크 인터페이스를 사용하여 요청합니다.

        * `NO_CACHE`: 쿼리는 캐시에서 초기 데이터를 반환 **하지 않습니다** . 대신 **항상** 네트워크 인터페이스를 사용하여 요청합니다. `NETWORK_ONLY` 정책과 달리 쿼리가 완료된 후 캐시에 데이터를 쓰지 않습니다.

          <OptionReference>
            NerdGraphQuery.FETCH_POLICY_TYPE.CACHE_AND_NETWORK, NerdGraphQuery.FETCH_POLICY_TYPE.CACHE_FIRST, NerdGraphQuery.FETCH_POLICY_TYPE.CACHE_ONLY, NerdGraphQuery.FETCH_POLICY_TYPE.NETWORK_ONLY, NerdGraphQuery.FETCH_POLICY_TYPE.NO_CACHE,
