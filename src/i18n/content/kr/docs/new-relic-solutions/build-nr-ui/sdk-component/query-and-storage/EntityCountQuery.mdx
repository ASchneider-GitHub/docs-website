각 엔터티도메인 및 엔터티 유형에 사용 가능한 엔터티 수를 쿼리합니다.

### 용법

```js
import { EntityCountQuery } from 'nr1'
```

### 예

#### 선언적 쿼리

```js
<EntityCountQuery>
  {({ loading, error, data }) => {
    if (loading) {
      return <Spinner />;
    }


    if (error) {
      return 'Error!';
    }


    return (
      <List items={data.types} rowCount={data.types.length} rowHeight={20}>
        {({ item }) => (
          <ListItem key={item.name}>
            {item.domain}|{item.type}: {item.count}
          </ListItem>
        )}
      </List>
    );
  }}
</EntityCountQuery>
```

#### 필수 쿼리

```js
EntityCountQuery.query().then(({ data }) => console.log(data));
```

      <td>
        `fetchPolicyType` <h5>열거</h5>
      </td>

      <td>
        쿼리가 캐시된 데이터와 상호 작용하는 방법을 지정할 수 있습니다.

        * `CACHE_AND_NETWORK`: 쿼리는 가능한 경우 캐시에서 초기 데이터를 반환합니다. 그러나 전체 데이터가 캐시에 있는지 여부에 관계없이 쿼리는 항상 네트워크 인터페이스를 사용하여 요청하고 업데이트된 데이터를 반환합니다. 구성요소의 정적 `query()` 메소드를 사용하는 경우에는 이 옵션을 사용할 수 없습니다.

        * `CACHE_FIRST`: 쿼리는 쿼리에 대한 데이터가 아직 캐시에 없는 경우 **에만** 네트워크 인터페이스를 사용하여 요청합니다.

        * `CACHE_ONLY`: 쿼리는 네트워크 인터페이스를 사용하여 요청 **하지 않습니다** . 대신 캐시에서 사용 가능한 데이터를 반환합니다. 쿼리에 대한 데이터가 캐시에 없으면 오류가 발생합니다.

        * `NETWORK_ONLY`: 쿼리는 캐시에서 초기 데이터를 반환 **하지 않습니다** . 대신 **항상** 네트워크 인터페이스를 사용하여 요청합니다.

        * `NO_CACHE`: 쿼리는 캐시에서 초기 데이터를 반환 **하지 않습니다** . 대신 **항상** 네트워크 인터페이스를 사용하여 요청합니다. `NETWORK_ONLY` 정책과 달리 쿼리가 완료된 후 캐시에 데이터를 쓰지 않습니다.

          <OptionReference>
            EntityCountQuery.FETCH_POLICY_TYPE.CACHE_AND_NETWORK, EntityCountQuery.FETCH_POLICY_TYPE.CACHE_FIRST, EntityCountQuery.FETCH_POLICY_TYPE.CACHE_ONLY, EntityCountQuery.FETCH_POLICY_TYPE.NETWORK_ONLY, EntityCountQuery.FETCH_POLICY_TYPE.NO_CACHE
          </OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `filters` <h5>문자열|(모양|모양|모양)\[]</h5>
      </td>

      <td>
        엔터티의 범위를 좁히는 데 사용되는 필터입니다. 필터에 대한 환불이며, 가능한 필터는 3개입니다:

        * 검색쿼리필터: `Object<type: string = "searchQuery", value: string>`
        * 엔터티 유형필터: `Object<type: string = "entityType", Object<domain: string, type: string>>`
        * 태그필터: `Object<type: string = "tag", Object<key: string, value: string>>`

        ```js
        const filters = [
          {
            type: 'searchQuery',
            value: 'foo',
          },
          {
            type: 'entityType',
            value: { domain: 'APM', type: 'APPLICATION' },
          },
          {
            type: 'tag',
            value: { key: 'environment', value: 'production' },
          },
          {
            type: 'tag',
            value: { key: 'team', value: 'bar' },
          },
        ];
        ```
      </td>
    </tr>
