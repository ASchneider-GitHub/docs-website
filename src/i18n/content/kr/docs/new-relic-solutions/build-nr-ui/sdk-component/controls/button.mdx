## 용법 [#usage]

```js
import { Button } from 'nr1'
```

## 예 [#examples]

### 기초적인 [#basic]

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>;
```

### 크기 [#sizes]

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.LARGE}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.MEDIUM}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.SMALL}>
    Click me
  </Button>
</div>;
```

### 유형 [#types]

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.NORMAL}>Click me</Button>
  <Button type={Button.TYPE.PRIMARY}>Click me</Button>
  <Button type={Button.TYPE.DESTRUCTIVE}>Click me</Button>
  <Button type={Button.TYPE.OUTLINE}>Click me</Button>
  <Button type={Button.TYPE.PLAIN}>Click me</Button>
  <Button type={Button.TYPE.PLAIN_NEUTRAL}>Click me</Button>
</div>;
```

### 비활성화된 상태 [#disabled-state]

```js
<div className="nr1-Docs-prettify">
  <Button disabled type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    disabled
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
    type={Button.TYPE.PRIMARY}
  >
    Click me
  </Button>
  <Button
    disabled
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
    type={Button.TYPE.PRIMARY}
  />
</div>;
```

### 로딩 상태 [#loading-state]

```js
<div className="nr1-Docs-prettify">
  <Button loading type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
    loading
    type={Button.TYPE.PRIMARY}
  >
    Click me
  </Button>
  <Button
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
    loading
    type={Button.TYPE.PRIMARY}
  />
</div>;
```

      <td>
        `ariaControls`
      </td>

      <td>
        끈
      </td>

      <td>
        패널을 확장하거나 여는 데 사용될 때 `Button` 가 제어하는 요소의 `id` 문자열을 전달합니다. `ariaExpanded` 와 함께 사용하세요.

        **버튼을 누르면 컨테이너가 확장됩니다.**

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
        <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
            Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
            Content
            </div>
        </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded`
      </td>

      <td>
        부울
      </td>

      <td>
        이를 사용하여 `Button` 가 제어하는 요소가 확장되는지 여부를 나타냅니다.

        `ariaControls` 및 `ariaHasPopup` 예시도 확인하는 것이 좋습니다.
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup`
      </td>

      <td>
        부울|문자열
      </td>

      <td>
        `Button` 가 대화형 오버레이 요소를 전환함을 나타내는 데 사용합니다. 값은 다음과 같습니다.

        * `false` (기본값)은 요소에 팝업이 없음을 나타냅니다.

        * `true` 팝업에 `menu` 역할이 있음을 나타냅니다.

        * `"menu"` 팝업에 `menu` 역할이 있음을 나타냅니다.

        * `"listbox"` 팝업에 `listbox` 역할이 있음을 나타냅니다.

        * `"tree"` 팝업에 `tree` 역할이 있음을 나타냅니다.

        * `"grid"` 팝업에 `grid` 역할이 있음을 나타냅니다.

        * `"dialog"` 팝업에 `dialog` 역할이 있음을 나타냅니다. 이 속성에 대한 [WAI ARIA 사양을](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup) 읽고 어느 것이 귀하의 사용 사례에 더 적합한지 알아보세요.

          **대화 상자를 열었음을 나타내는 버튼**

          ```js
          const { modalHidden } = this.state;

          return (
          <>
              <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
              >
              Open modal
              </Button>
              <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
              />
          </>
          );
          ```
      </td>
    </tr>

        **접근성이 향상된 아이콘 전용 버튼**

        ```js
        <Button

            ariaLabel="Search for entities"
        />;
        ```

        슬림한 버튼은 버튼의 중요성을 감소시키므로 아껴서 사용하세요. 공간에 맞추기 위해서만 사용하지 말고 기본 버튼 주위의 공간을 늘리는 것을 고려하십시오.

        큰 버튼을 사용하는 몇 가지 사례는 추가 기능에 대한 마케팅과 유사한 자료(스플래시 페이지 또는 영웅 메시지에 소개)입니다.

        `data-test-id` 속성을 추가합니다. 이를 사용하여 단위 및 E2E 테스트의 구성 요소를 목표로 삼습니다. 테스트 ID가 유효하려면 nerdpack ID 접두어를 붙이고 그 뒤에 점이 옵니다. 예를 들어, `my-nerdpack.some-element`.

        **참고**: `data-test-id` 속성이 DOM 에서 제거될 때 표시되지 않을 수 있습니다. 이를 디버그하려면 `e2e-test` 쿼리 변수를 URL에 전달해야 합니다.

        유형

        열거

      <td>
        기본값: `Button.TYPE.NORMAL`. 유형은 다음과 같습니다.

        * 주요한. 사용자가 다음 단계로 수행하기를 원하는 하나의 특정 작업에 주의를 환기시키는 데 사용됩니다.

        * 일반(보조로 사용) - 여러 작업을 표시해야 할 때 사용합니다.

        * 개요. 지정된 뷰의 기본(기본) 작업이 아닌 중요한 작업에 사용합니다. 이 변형은 계층 구조 측면에서 기본 버튼과 기본 버튼 사이에 위치합니다.

        * 솔직한. 사용자에게 덜 중요한 여러 작업을 수행해야 할 때 사용합니다.

        * 파괴적이다. 사용자가 완료하기 전에 일시 중지하고 고려하도록 하는 삭제 또는 제거와 같은 파괴적인 작업이 있는 경우에 사용하세요.

          ```
          <One of
              Button.TYPE.DESTRUCTIVE,
              Button.TYPE.NORMAL,
              Button.TYPE.OUTLINE,
              Button.TYPE.PLAIN,
              Button.TYPE.PLAIN_NEUTRAL,
              Button.TYPE.PRIMARY,
          >
          ```
