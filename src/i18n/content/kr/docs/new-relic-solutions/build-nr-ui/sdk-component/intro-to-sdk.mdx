뉴렐릭 앱을 구축하는 데 도움이 되도록 뉴렐릭 SDK 구성요소를 제공합니다.

여기서는 SDK의 API 호출 및 구성 요소 유형을 소개합니다. SDK는 Nerdlet을 구축하고, 시각화를 생성하고, 뉴렐릭 또는 타사 데이터를 가져오는 데 필요한 모든 것을 제공합니다.

## SDK의 구성 요소 [#components]

[뉴렐릭 CLI를](/docs/new-relic-solutions/tutorials/new-relic-cli/#install-the-new-relic-cli) 설치할 때 얻는 `nr1` 노드 모듈 패키지에서 SDK 구성 요소를 찾을 수 있습니다. `nr1` 구성요소는 여러 범주로 구분됩니다.

* UI 구성요소
* 차트 구성요소
* 쿼리 및 저장소 구성 요소
* 플랫폼 API

### UI 구성요소 [#ui-components]

SDK의 **UI components** \[UI 구성 요소] 카테고리에는 다음을 포함한 React UI 구성 요소가 포함되어 있습니다.

* **텍스트 구성 요소**: 이 구성 요소는 기본 글꼴 및 제목 요소를 제공합니다. 여기에는 `HeadingText` 및 `BlockText` 이 포함됩니다.

* **레이아웃 구성 요소**: 이러한 구성 요소를 사용하면 레이아웃을 제어할 수 있으며 CSS를 다루지 않고도 복잡한 레이아웃 디자인을 구축할 수 있습니다. 레이아웃 구성요소에는 다음이 포함됩니다.

  * `Grid` 및 `GridItem`: 더 복잡하고 더 큰 규모의 페이지 콘텐츠를 행과 열로 구성합니다.
  * `Stack` 및 `StackItem`: 열이나 행으로 더 간단하고 작은 규모의 페이지 콘텐츠를 구성합니다.
  * `Tabs` 및 `TabsItem`: 다양한 관련 콘텐츠를 숨길 수 있는 별도의 섹션으로 그룹화합니다.
  * `List` 및 `ListItem`: 가상화된 목록의 기본 골격을 제공합니다.
  * `Card`, `CardHeader` 및 `CardBody`: 유사한 개념과 작업을 그룹화하는 데 사용됩니다.

* **양식 구성 요소**: 이 구성 요소는 UI와 상호 작용하기 위한 기본 구성 요소를 제공합니다. 여기에는 [`Button`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/button), [`TextField`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/text-field), [`Dropdown`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown), [`DropdownItem`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown-item), [`Checkbox`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/checkbox), [`RadioGroup`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio-group) 및 [`Radio`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio) 이 포함됩니다.

* **피드백 구성 요소**: 이러한 구성 요소를 사용하여 사용자가 수행한 작업에 대한 피드백을 제공합니다. 여기에는 `Spinner` 및 `Toast` 이 포함됩니다.

* **오버레이된 구성 요소**: 이러한 구성 요소를 사용하면 작업이나 이벤트가 트리거될 때 화면의 다른 콘텐츠 위에 나타나는 추가 하위 보기 형태로 상황별 정보와 옵션을 표시할 수 있습니다. 사용자 복합 복합체(모달과 같은)를 요구하거나 (툴팁과 같은) 기능을 강화할 수 있습니다. 여기에는 `Modal` 및 `Tooltip` 이 포함됩니다.

<Callout variant="important">
  접미사가 `Item` 인 구성요소는 접미사가 없는 해당 이름의 직계 하위 항목으로만 작동할 수 있습니다. 예를 들어, `GridItem` `Grid` 의 하위 항목으로만 발견되어야 합니다.
</Callout>

### 차트 구성요소 [#chart-components]

SDK의 **Charts** \[차트] 카테고리에는 다양한 유형의 차트를 나타내는 구성 요소가 포함되어 있습니다. `ChartGroup` 구성요소는 관련 차트 그룹이 데이터를 공유하고 정렬하는 데 도움이 됩니다.

일부 차트 구성 요소는 자체적으로 [NRQL 쿼리를](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/) 수행할 수 있습니다. 일부는 사용자 정의된 데이터 세트를 허용합니다.

### 쿼리 및 저장소 구성 요소 [#query-storage]

**Query components** \[쿼리 구성 요소] 범주에는 뉴렐릭 데이터를 가져오고 저장하는 구성 요소가 포함되어 있습니다.

데이터를 가져오는 주요 방법은 GraphQL 엔드포인트인 NerdGraph를 사용하는 것입니다. 이는 `NerdGraphQuery` 사용하여 쿼리할 수 있습니다. NerdGraph 쿼리 사용을 단순화하기 위해 사전 정의된 쿼리가 포함된 일부 구성 요소를 제공합니다. 자세한 내용은 [NerdGraph](/collect-data/get-started-nerdgraph-api-explorer) 에 대한 문서를 참조하세요.

또한 구성 설정 데이터 또는 사용자별 데이터와 같은 소규모 데이터 세트를 저장하기 위한 스토리지도 제공합니다. 이에 대한 자세한 내용은 [NerdStorage를](/docs/new-relic-solutions/build-nr-ui/nerdstorage) 참조하세요.

### 플랫폼 API

SDK의 플랫폼 API 구성 요소를 사용하면 URL에서 상태를 읽고 쓰고 설정 등을 설정하여 뉴렐릭 플랫폼의 다양한 부분과 상호 작용할 수 있습니다. 이는 다음 범주로 나뉩니다.

* `PlatformStateContext`: 플랫폼 URL 상태 변수에 대한 읽기 액세스를 제공합니다. 예를 들어 [타임 피커](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) 의 `timeRange` .
* `navigation`: 뉴렐릭에서 탐색을 프로그래밍 방식으로 조작할 수 있는 개체입니다. 예: 새 Nerdlet 열기.
* `NerdletStateContext`: Nerdlet URL 상태 변수에 대한 읽기 액세스를 제공합니다. 예를 들어 [엔터티 탐색기](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/) 의 `entityGuid` .
* `nerdlet`: Nerdlet을 구성하고 Nerdlet의 URL 상태에 쓸 수 있습니다.
