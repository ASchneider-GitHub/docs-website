[CloudWatch アプリケーション Signals は、](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Agent-Application_Signals.html) AWSホストするサービスを監視するためにAWSが提供するオプションです。 データは、CloudWatch Metric Streams を通じてNew Relicで利用できます。

## 統合機能の有効化 [#activate]

この統合を有効にするには、 [CloudWatch メトリック ストリームをNew Relicに設定する](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを見つけて使用する [#find-use-data]

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &amp;gt; Metrics and events**</DNT>に移動して`aws.applicationsignals`でフィルタリングします。

### アラートの作成 [#create-alerts]

変更があった場合に通知するように<InlinePopover type="alerts" />を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

アラートの作成について詳しくは、[こちら](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)をご覧ください。
