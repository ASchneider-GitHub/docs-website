表のヘッダーにある表セルの内容を上部に保持し、必要に応じて追加のコントロールを追加することで折り返します。例: ソートメカニズム。

### 使用方法

```js
import { TableHeaderCell } from 'nr1'
```

        <FunctionDefinition
          returnValue={[]}
          arguments={[{"name":"event","type":"React.MouseEvent","description":""},{"name":"sortingData","type":"Object","description":""}]}

      <td>
        `sortable` <h5>ブール値</h5>
      </td>

      <td>
        列が並べ替え可能かどうか。 これは、見出しテキストの横に上矢印と下矢印が表示されることを意味します。ソートは制御されたコンポーネント インターフェースであり、ソート イベントをリッスンし、 `sortingOrder`プロパティを変更して対応する列にソートを適用する責任があります。
      </td>
    </tr>

    <tr>
      <td>
        `sortingFunction` <h5>機能</h5>
      </td>

      <td>
        行をソートする方法。 デフォルトのメソッドは通常、適切な処理 (テキストの場合はアルファベット順、数字の場合は数値順、ブール値のグループ化) を行いますが、必要に応じてオーバーライドできます。この関数は 2 つのパラメーター ( `a`と`b` ) を受け取り、 `a`が`b`の前に出現すると見なされる場合は負の数を返し、後に出現すると見なされる場合は正の数を返し、両方の値が同一の場合は 0 を返す必要があります。コールバックに渡される各オブジェクトには、指定された行の値 (つまり、行に対して`value`を呼び出した結果)、行項目 ( `item` )、行のインデックス ( `index` )、および`items`配列内のすべての項目 ( `items` ) が含まれます。 通常は、 `a.item`と`b.item`を使用して行を比較します。

        <FunctionDefinition
          returnValue={[{"type":"number","description":""}]}
          arguments={[{"name":"a","type":"object","description":""},{"name":"b","type":"object","description":""}]}
        />
      </td>
    </tr>

    <tr>
      <td>
        `sortingOrder` <h5>ナンバー</h5>
      </td>

      <td>
        並べ替え順序を確立します。 最も小さい数字は最高のソート優先順位を示します。つまり、 `0`最初にソートされる列になり、2 つの項目が同一の場合は列`1`が使用されるということになります。 明示的なソート順序を持たない列は、明示的なソート順序を持つすべての列の後に使用され、表示順に適用されます。
      </td>
    </tr>

        </OptionReference>

        `data-test-id`属性を追加します。 ユニット テストと E2E テストでコンポーネントをターゲットにするために使用します。テスト ID を有効にするには、nerdpack ID をプレフィックスとして付け、その後にドットを付けます。たとえば、 `my-nerdpack.some-element` 。

        **注:** `data-test-id` 属性はDOMから削除されているため表示されない場合があります。デバッグするには、URL に `e2e-test` クエリ引数を渡します。

      <td>
        セルのタイトルを設定します。 これは、列の幅が狭くて見出し全体を表示できない場合に、ツールチップの目的で使用されます。 デフォルトでは、提供された子の文字列化されたバージョンになります。
      </td>
    </tr>

    <tr>
      <td>
        `value` <h5>機能</h5>
      </td>

      <td>
        列を表す生の値を抽出するために使用される関数。 生の値は、ブール値 (チェックボックスをレンダリングする場合など)、数値 (メトリックをレンダリングする場合)、またはその他のテキストになります。この関数は、行を表す項目、テーブルに渡される`items`配列内のそのインデックス、および項目自体を含むオブジェクトを使用して呼び出されます。この関数は、テーブルを (指定された値で) 並べ替えるだけでなく、 `'fit-content'`幅のヒューリスティックにも使用されます。
      </td>
    </tr>

    <tr>
      <td>
        `width` <h5>ストリング</h5>
      </td>

      <td>
        列の幅を表します。 これは常に文字列であり、次のいずれかの型になります。

        * `'fit-content'`: 列内のすべての項目を割り当てるために必要な最小限のスペースが列に使用されます。 仮想化されたリストの場合、すべてのセルがレンダリングされないようにヒューリスティックが使用されます。
        * `'YYpx'`: `YY`は数値で、列の幅をピクセル単位で表します。
        * `'YY%'`: `YY`は数値で、テーブルがレンダリングされる使用可能なスペースのサイズに対する列の幅をパーセンテージで表します。
        * `'YYfr'`: ここで、 `YY`は数値であり、使用可能な幅の割合を表します。 分数は他の分数に比例します。たとえば、2 つの列 ( `1fr`と`2fr` ) を持つテーブルでは、2 番目の列は最初の列の 2 倍の大きさになります。幅を計算した後、テーブル全体のサイズが使用可能なスペースをカバーしない場合は、すべての列が使用可能なスペースに収まるように比例して拡大されます。
      </td>
    </tr>
