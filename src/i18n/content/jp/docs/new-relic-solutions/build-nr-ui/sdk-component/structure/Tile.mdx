### 使用方法

```js
import { Tile } from 'nr1'
```

### 例

#### 基本タイル

```js
<Tile onClick={console.log}>
  <HeadingText type={HeadingText.TYPE.HEADING_6}>
    Single location failure
  </HeadingText>
  <BlockText>
    Check your company’s style guides. Great copy that doesn’t fit with the rest
    of the interface or marketing materials can be jarring to users.
  </BlockText>
</Tile>
```

#### 無効なタイル

```js
<Tile onClick={console.log} disabled>
  <HeadingText type={HeadingText.TYPE.HEADING_6}>
    Single location failure
  </HeadingText>
  <BlockText>
    Check your company’s style guides. Great copy that doesn’t fit with the rest
    of the interface or marketing materials can be jarring to users.
  </BlockText>
</Tile>
```

#### リンクタイル

```js
<Tile to="https://login.newrelic.com/login">Login</Tile>
```

### 小道具

<table>

      <td>
        これを使用して、スクリーン リーダーを使用するユーザーに対してコンポーネントのアクションのコンテキストをより適切に説明します。`<Tile to="/dashboards" ariaLabel="Go to your dashboard"> Dashboards </Tile>`
      </td>
    </tr>

    <tr>
      <td>
        `children` <h5>必須</h5><h5>ノード</h5>
      </td>

      <td>
        コンポーネント内でレンダリングするコンテンツ。
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>ストリング</h5>
      </td>

      <td>
        コンポーネントにクラス名を追加します。配置と間隔の目的にのみ使用してください。
      </td>
    </tr>

    <tr>
      <td>
        `disabled` <h5>ブール値</h5>
      </td>

      <td>
        `true`の場合、タイルはインタラクションに使用できません。
      </td>
    </tr>

    <tr>
      <td>
        `onClick` <h5>機能</h5>
      </td>

      <td>
        ユーザーがタイルをクリックするたびにコールバックが起動されます。<FunctionDefinition returnValue={[]} arguments={[{"name":"event","type":"React.MouseEvent","description":""}]}/>
      </td>
    </tr>

    <tr>
      <td>
        `sizeType` <h5>enum</h5>
      </td>

      <td>
        タイルの希望サイズを指定します (サイズが小さいほど内部のパディングが少なくなります) <OptionReference>。Tile.SIZE_TYPE.MEDIUMTile.SIZE_TYPE.SMALL</OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `spacingType` <h5>列挙型\[]</h5>
      </td>

      <td>
        間隔プロパティ。 間隔は、0 から 4 個の値のタプルとして定義され、 `margin`や`padding`などの CSS プロパティと同じ規則に従います。 値を省略するには、 `SPACING_TYPE.OMIT`を使用します<OptionReference array>。Tile.SPACING_TYPE.EXTRA_LARGETile.SPACING_TYPE.LARGETile.SPACING_TYPE.MEDIUMTile.SPACING_TYPE.NONETile.SPACING_TYPE.OMITTile.SPACING_TYPE.SMALL</OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>物体</h5>
      </td>

      <td>
        カスタム スタイル設定用のインライン スタイル。配置と間隔設定の目的でのみ使用してください。
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>ストリング</h5>
      </td>

      <td>
        `data-test-id`属性を追加します。 ユニット テストおよび E2E テストでコンポーネントを ターゲット するために使用します。テスト ID を有効にするには、nerdpack ID をプレフィックスとして付け、その後にドットを付けます。たとえば、`my-nerdpack.some-element` のようになります。**注:** `data-test-id` 属性はDOMから削除されているため表示されない場合があります。デバッグするには、URL に `e2e-test` クエリ引数を渡します。
      </td>
    </tr>

    <tr>
      <td>
        `to` <h5>形状|文字列</h5>

        リンク先の場所オブジェクトまたは URL 文字列。

        <h3>形</h3>

        `pathname` <h5>必須</h5><h5>文字列</h5>

        `search` <h5>ストリング</h5>

        `hash` <h5>ストリング</h5>

      <td>
        `type` <h5>enum</h5>

        * ソリッド - タイルを白い背景と対比させる必要がある場合に使用します。

          <OptionReference>
            タイル.TYPE.アウトライン、タイル.TYPE.プレーン、タイル.TYPE.ソリッド、

      </td>
