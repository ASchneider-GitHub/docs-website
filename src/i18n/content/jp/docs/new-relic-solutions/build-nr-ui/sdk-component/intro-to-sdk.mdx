New Relic アプリケーションの構築を支援するために、New Relic アプリの構築に使用される New Relic SDK コンポーネントを提供しています。

ここでは、SDK の API コールとコンポーネントの種類について紹介します。 SDK には、Nerdlet の構築、視覚化の作成、New Relic またはサードパーティのデータの取得に必要なものがすべて揃っています。

## SDKのコンポーネント [#components]

SDK コンポーネントは、 [New Relic CLI を](/docs/new-relic-solutions/tutorials/new-relic-cli/#install-the-new-relic-cli)インストールすると取得される`nr1`という名前の Node モジュール パッケージにあります。 `nr1`コンポーネントはいくつかのカテゴリに分かれています:

* UI コンポーネント
* チャートの構成要素
* クエリとストレージコンポーネント
* プラットフォームAPI

### UI コンポーネント [#ui-components]

SDK の**UI components** \[UI コンポーネント]カテゴリには、次のような React UI コンポーネントが含まれています。

* **テキスト コンポーネント**: これらのコンポーネントは、基本的なフォントと見出し要素を提供します。 これらには`HeadingText`と`BlockText`が含まれます。

* **レイアウト コンポーネント**: これらのコンポーネントを使用すると、レイアウトを制御でき、CSS を扱わずに複雑なレイアウト デザインを構築できます。 レイアウト コンポーネントには次のものが含まれます。

  * `Grid` `GridItem` : より複雑で大規模なページ コンテンツを行と列に整理します。
  * `Stack` `StackItem` : よりシンプルで小規模なページ コンテンツを列または行に整理します。
  * `Tabs` および`TabsItem` : 関連するさまざまなコンテンツを個別の非表示可能なセクションにグループ化します。
  * `List` `ListItem` : 仮想化されたリストの基本的な骨組みを提供します。
  * `Card`、 `CardHeader` 、 `CardBody` : 類似した概念やタスクをグループ化するために使用されます。

* **フォーム コンポーネント**: これらのコンポーネントは、UI と対話するための基本的な構成要素を提供します。 これらには、 [`Button`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/button) 、 [`TextField`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/text-field) 、 [`Dropdown`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown) 、 [`DropdownItem`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/dropdown-item) 、 [`Checkbox`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/checkbox) 、 [`RadioGroup`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio-group) 、 [`Radio`](/docs/new-relic-solutions/build-nr-ui/sdk-component/controls/radio)が含まれます。

* **フィードバック コンポーネント**: これらのコンポーネントを使用して、ユーザーが実行したアクションに関するフィードバックをユーザーに提供します。 これらには`Spinner`と`Toast`含まれます。

* **オーバーレイ コンポーネント**: これらのコンポーネントを使用して、アクションまたはイベントがトリガーされたときに画面上の他のコンテンツの上に表示される追加の子ビューの形式でコンテキスト情報とオプションを表示します。 これらは、ユーザー インタラクション を必要とするもの（モーダルなど）や、拡張するもの（ツールチップなど）として使用できます。 これらには`Modal`と`Tooltip`含まれます。

<Callout variant="important">
  `Item`という接尾辞が付いたコンポーネントは、接尾辞のないその名前の直接の子としてのみ動作できます。 たとえば、 `GridItem` `Grid`の子としてのみ見つかります。
</Callout>

### チャートの構成要素 [#chart-components]

SDK の**Charts** \[チャート]カテゴリには、さまざまな種類のチャートを表すコンポーネントが含まれています。 `ChartGroup`コンポーネントは、関連するグラフのグループがデータを共有し、整列させるのに役立ちます。

一部のチャート コンポーネントは独自に[NRQL クエリ](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/)を実行できますが、カスタマイズされたデータ セットを受け入れるものもあります。

### クエリとストレージコンポーネント [#query-storage]

**Query components** \[クエリ コンポーネント]カテゴリには、New Relic データを取得して保存するためのコンポーネントが含まれています。

データを取得する主な方法は、GraphQL エンドポイントである NerdGraph を使用することです。 これは`NerdGraphQuery`を使用してクエリできます。 NerdGraph クエリの使用を簡素化するために、事前定義されたクエリを備えたコンポーネントをいくつか提供しています。 詳細については、 [NerdGraph](/collect-data/get-started-nerdgraph-api-explorer)に関するドキュメントをご覧ください。

構成設定データやユーザー固有のデータなどの小さなデータセットを保存するためのストレージも提供しています。 詳細については、 [NerdStorage](/docs/new-relic-solutions/build-nr-ui/nerdstorage)を参照してください。

### プラットフォームAPI

SDK のプラットフォームAPIコンポーネントを使用すると、URL からの状態の読み取りや URL への状態の書き込み、設定などを行うことで、アプリケーションがNew Relic Platform のさまざまな部分と対話できるようになります。 これらは以下のカテゴリーに分類されます:

* `PlatformStateContext`: プラットフォーム URL 状態変数への読み取りアクセスを提供します。 たとえば[、タイムピッカー](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)の`timeRange`などです。
* `navigation`: New Relic のナビゲーションをプログラムで操作できるようにするオブジェクト。 例: 新しい Nerdlet を開く。
* `NerdletStateContext`: Nerdlet URL 状態変数への読み取りアクセスを提供します。 たとえば、 [エンティティ エクスプローラー](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/)の`entityGuid`などです。
* `nerdlet`: Nerdlet を設定し、Nerdlet の URL 状態に書き込むことができます。
