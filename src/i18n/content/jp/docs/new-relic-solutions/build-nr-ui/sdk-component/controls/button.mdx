## 使用方法 [#usage]

```js
import { Button } from 'nr1'
```

## 例 [#examples]

### ベーシック [#basic]

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>;
```

### サイズ [#sizes]

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.LARGE}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.MEDIUM}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.SMALL}>
    Click me
  </Button>
</div>;
```

### 種類 [#types]

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.NORMAL}>Click me</Button>
  <Button type={Button.TYPE.PRIMARY}>Click me</Button>
  <Button type={Button.TYPE.DESTRUCTIVE}>Click me</Button>
  <Button type={Button.TYPE.OUTLINE}>Click me</Button>
  <Button type={Button.TYPE.PLAIN}>Click me</Button>
  <Button type={Button.TYPE.PLAIN_NEUTRAL}>Click me</Button>
</div>;
```

        `ariaControls`
      </td>

      <td>
        ストリング
      </td>

      <td>
        パネルを展開または開くために使用されるときに、 `Button`が制御する要素の`id`文字列を渡します。 `ariaExpanded`と一緒に使用してください。

        **ボタンはコンテナを展開します**

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
        <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
            Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
            Content
            </div>
        </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded`
      </td>

      <td>
        ブール値
      </td>

      <td>
        `Button`が制御する要素が展開されているかどうかを示すために使用します。

        `ariaControls`および`ariaHasPopup`の例も確認することをお勧めします。
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup`
      </td>

      <td>
        ブール値|文字列
      </td>

      <td>
        `Button`がインタラクティブなオーバーレイ要素を切り替えることを示すために使用します。 値は次のとおりです。

        * `false` (デフォルト) 要素にポップアップがないことを示します。

        * `true` ポップアップに`menu`役割があることを示します。

        * `"menu"` ポップアップに`menu`役割があることを示します。

        * `"listbox"` ポップアップに`listbox`役割があることを示します。

        * `"tree"` ポップアップに`tree`役割があることを示します。

        * `"grid"` ポップアップに`grid`役割があることを示します。

        * `"dialog"` ポップアップに`dialog`役割があることを示します。 この属性の[WAI ARIA 仕様を](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup)読んで、どの属性が使用ケースに適しているかを確認してください。

          **ダイアログを開くことを示すボタン**

          ```js
          const { modalHidden } = this.state;

          return (
          <>
              <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
              >
              Open modal
              </Button>
              <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
              />
          </>
          );
          ```
      </td>
    </tr>

        `sizeType`
      </td>

      <td>
        enum
      </td>

      <td>
        デフォルト値: `Button.SIZE_TYPE.MEDIUM` 。 ほとんどすべてのインスタンスで通常サイズのボタンを使用することをお勧めします。

        スリムボタンはボタンの重要性を低下させるため、控えめに使用してください。 スペースに収めるためだけに使用せず、デフォルト ボタンの周囲のスペースを増やすことを検討してください。

        大きなボタンを使用する数少ないケースは、アドオンのマーケティングのような素材、つまりスプラッシュ ページやヒーロー メッセージでの紹介です。

        ```
        <One of
            Button.SIZE_TYPE.LARGE,
            Button.SIZE_TYPE.MEDIUM,
            Button.SIZE_TYPE.SMALL,
        >
        ```
      </td>
    </tr>

        ```
        <Array of
            <One of
                Button.SPACING_TYPE.EXTRA_LARGE,
                Button.SPACING_TYPE.LARGE,
                Button.SPACING_TYPE.MEDIUM,
                Button.SPACING_TYPE.NONE,
                Button.SPACING_TYPE.OMIT,
                Button.SPACING_TYPE.SMALL,
            >
        >
        ```

    <tr>
      <td>
        タイプ
      </td>

      <td>
        enum
      </td>

      <td>
        デフォルト値: `Button.TYPE.NORMAL` 。 タイプは次のようになります:

        * 主要な。 次のステップとしてユーザーに実行してほしい特定のアクションに注意を喚起するために使用します。

        * 通常 (セカンダリとして使用) - 複数のアクションを表示する必要がある場合に使用します。

        * 概要。 特定のビューのメイン (プライマリ) アクションではない重要なアクションに使用します。 このバリエーションは、階層的にはプライマリ ボタンとデフォルト ボタンの間に位置します。

        * 無地。 ユーザーにとってそれほど重要ではない複数のアクションを実行する必要がある場合に使用します。

        * 破壊的。 削除や除去などの破壊的なアクションがあり、完了する前にユーザーに一時停止して検討してもらいたい場合に使用します。

          ```
          <One of
              Button.TYPE.DESTRUCTIVE,
              Button.TYPE.NORMAL,
              Button.TYPE.OUTLINE,
              Button.TYPE.PLAIN,
              Button.TYPE.PLAIN_NEUTRAL,
              Button.TYPE.PRIMARY,
          >
          ```
      </td>
    </tr>
