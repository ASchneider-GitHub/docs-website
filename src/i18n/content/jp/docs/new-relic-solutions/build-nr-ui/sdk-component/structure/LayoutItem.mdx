`<Layout>`コンポーネントの子要素。

デフォルトでは、レイアウト領域の全幅に渡るメイン タイプが使用されます。 左に分割と右に分割すると、固定された左/右領域が提供されます。 レイアウトはそれぞれ 1 つのタイプのみをサポートします。

### 使用方法

```js
import { LayoutItem } from 'nr1'
```

      </td>

      <td>
        レイアウト項目の幅を決定します。 ナビゲーションの目的では小さいサイズを使用することをお勧めします (例: サイドバーなど) と、コンテキスト情報を表示するための中サイズ (例: アクティビティ ストリーム)。サイズ タイプは SPLIT_LEFT および SPLIT_RIGHT アイテムでのみ機能します。

        <OptionReference>
          レイアウトアイテム.SIZE_TYPE.MEDIUM、レイアウトアイテム.SIZE_TYPE.SMALL、
        </OptionReference>
      </td>
    </tr>

      <td>
        `style` <h5>物体</h5>
      </td>

      <td>
        カスタム スタイル用のインライン スタイル。
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>ストリング</h5>
      </td>

      <td>
        `data-test-id`属性を追加します。 ユニット テストと E2E テストでコンポーネントをターゲットにするために使用します。テスト ID を有効にするには、nerdpack ID をプレフィックスとして付け、その後にドットを付けます。たとえば、 `my-nerdpack.some-element` 。

        **注:** `data-test-id` 属性はDOMから削除されているため表示されない場合があります。デバッグするには、URL に `e2e-test` クエリ引数を渡します。
      </td>
    </tr>

    <tr>
      <td>
        `type` <h5>enum</h5>
      </td>

      <td>
        タイプは次のようになります:

        * メイン - 各ビューにメインコンテンツを配置するために使用されます。 レイアウト領域の全幅に広がります。

        * 左に分割 - レイアウトの左側の固定領域としてレンダリングします。

        * 右に分割 - レイアウトの右側の固定領域としてレンダリングします。

          <OptionReference>
            レイアウト項目.TYPE.MAIN、レイアウト項目.TYPE.SPLIT_LEFT、レイアウト項目.TYPE.SPLIT_RIGHT、
          </OptionReference>
